# endless session goto / back — tmux session navigation

## Goal
Make any session (notably the session-next view's `⟳` in-flight rows) directly navigable:
- `endless session goto <task|session>` — switch tmux focus to the target session's pane.
- `endless session back` — return to where you came from (browser-style).

## `session goto <id>`
- `<id>`: a task id (`E-NNNN` or `NNNN`) or a session id.
- Resolve to a target tmux pane:
  - session id → that session's `sessions.process` (the pane, e.g. `%214`).
  - task id → the live session whose `active_task_id` = that task; if several, the most-recently-active (`last_activity DESC`); if none live, error: "no live session on E-NNNN".
- Push the CURRENT pane (`$TMUX_PANE`) onto the back-stack BEFORE switching.
- Switch with `tmux switch-client -t <pane>` (tmux resolves pane → its window/session). Friendly error if not inside tmux, or if the target pane no longer exists.

## `session back`
- Pop the top of the back-stack and `tmux switch-client -t <pane>`.
- Empty stack → message: "no previous session".
- If the popped pane is gone, drop it and pop the next.

## Back-stack
- A per-tmux-server stack of pane ids. Push on `goto`, pop on `back`.
- Storage: a tmux user option (e.g. `@endless_session_backstack`, space-separated) — simplest, dies with the server — or a small state file under `~/.cache/endless/`. Implementer's choice.

## Implementation notes
- Go (reads-in-Go rule); both verbs slot into the existing `endless session` command group.
- Reuse the session-resolution that `endless session cd` (E-990) already does; here we need session→pane via `sessions.process` rather than session→worktree path.
- Guard rails: not-in-tmux → friendly error; stale/closed target pane → error and drop it from the stack.

## Out of scope
- The `session next` view itself (E-1465) — separate task.
