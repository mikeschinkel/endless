# E-1566 ‚Äî Add per-type handoff variants for task/bug/research/epic

Lands after **E-1565** (Go template renderer). Splits the single `templates/handoff.md.tmpl` into four per-type templates and teaches Python to select by `tasks.type_id` slug.

## Decisions

- **Directory layout.** `templates/handoff/{task,bug,research,epic}.md.tmpl` ‚Äî one per slug. Old `templates/handoff.md.tmpl` deleted (replaced by `templates/handoff/task.md.tmpl` which inherits today's content).
- **Renderer takes path-style name.** `endless-go template render handoff/<type>`. The embedded FS preserves the directory; the verb's argument is the path under `templates/` without `.tmpl`. Same renderer code as E-1565 ‚Äî only the embed pattern broadens (`templates/**/*.tmpl`).
- **Type resolution in Python, not Go.** `render_handoff(..., task_type: str | None)` joins `tasks.type_id` ‚Üí `task_types.slug` at call sites. Unknown/null ‚Üí `"task"` fallback (matches design "Unknown/null type falls back to task"). Renderer in Go stays type-agnostic.
- **Child-count is universal.** Per E-1552 outcome ("All handoffs include the child count and recommend `--children`"), `render_handoff` runs `SELECT count(*) FROM tasks WHERE parent_id = ?` and injects `{{.child_count}}`. Templates render a conditional line: `{{if gt .child_count 0}}This task has {{.child_count}} children ‚Äî read them with `endless task show E-{{.spawned_id}} --children`.{{end}}`. Zero children = line absent.
- **Children-state breakdown deferred.** That richer payload comes in **E-1567**. This child ships `epic.md.tmpl` with a TODO marker comment (`{{/* E-1567: inject children-state breakdown here */}}`) so the renderer doesn't error and the implementer of E-1567 has a clear insertion point.
- **Per-variant framing.** Each template starts from `handoff/task.md.tmpl` (the verbatim today-content) and adds 1‚Äì3 lines of type-specific framing near the top:
  - **task** ‚Äî verbatim, no addition. End-state: `verify`. (Today's content.)
  - **bug** ‚Äî adds: *"Reproduce the bug first before changing anything. Link the failing test or the exact reproduction steps in your first reply."* End-state: `verify`.
  - **research** ‚Äî adds: *"Findings are the deliverable. Write them into `tasks.outcome` via `endless task update E-X --outcome <file>`. End state is `completed` (research has its own gate, ED-1502), not `verify`."* Adjusts step 6 to instruct `--status completed --outcome <file>` instead of `--status verify`.
  - **epic** ‚Äî adds: *"You're the coordinator on this epic. Don't implement directly ‚Äî draft plans, dispatch children, verify their work."* Adjusts step 3 to point at children (`endless task show E-X --children`) and replaces step 6 with epic-completion guidance (`endless task update E-X --status completed` once all children are `confirmed`/`assumed`). Plus the E-1567 insertion marker.
