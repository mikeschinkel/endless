# E-1568 — Add --bg flag to endless task spawn that dispatches via claude --bg --name carrier

Lands after **E-1565** (renderer), **E-1566** (type variants), **E-1571** (sessions: kind + active_epic_id), and **E-1569** (bgIsolation:'none' in settings). E-1571 shipped only the first two columns; the additional bg-row schema (short_id, nullable session_id) lands inside this child. This is the centerpiece of the epic.

## Decisions

### CLI surface
- New flag `--bg` on `endless task spawn <id>`.
- `--worktree <path>` works in both fg and bg paths: in bg, it sets `subprocess.run(..., cwd=<path>)` so the bg agent inherits a non-default cwd. Default remains the spawn-created worktree at `.endless/worktrees/e-<id>/`. (Not mutually exclusive — the only operational caveat is that a bg agent's filesystem context is invisible until you `attach`; if you set `--worktree` and never attach, the override is silent. Harmless.)
- Compatible with `--reopen`, `--force`, `--no-plan` (the last is a no-op — bg agents don't have a `/plan` slash-key concept; document and ignore).
- Compatible with `--type` of the target task (renderer picks the right handoff variant).

### Invocation
Per `docs/research-2026-06-12-claude-background-agents.md` §2/§7:
```bash
claude --bg --name "E-<task_id>" "<rendered-handoff-text>"
```
Prompt is positional argv (handoff is ~2 KB — well under Darwin's ARG_MAX of 256 KB). Stdout returns:
```
backgrounded · <8hex> · E-<task_id>
  claude attach <8hex>    open in this terminal
  ...
```

### Short-ID capture
Parse the first stdout line matching `^backgrounded · ([a-f0-9]+) · `. Regex pinned in code with comment + research-doc citation. On no-match, raise `ClickException` with the raw stdout — never silently proceed with a missing id.

### Session row write
At pre-claim time (BEFORE invoking claude), spawn already creates the worktree and flips status to `in_progress`. After the `claude --bg` subprocess returns and short-id is parsed, INSERT into `sessions`:

| Column | Value |
|---|---|
| `session_id` | **NULL** — bg agents have no UUID at dispatch. The bg agent's `SessionStart` hook UPDATEs this column with the real UUID once it fires (see Hook update below). SQLite's UNIQUE constraint treats NULL as distinct, so multiple pending bg rows coexist. (`session_id NOT NULL` relaxed by this child's schema-change file.) |
| `project_id` | resolved from task row |
| `platform` | `'claude'` |
| `state` | `'working'` |
| `active_task_id` | `<task_id>` |
| `active_epic_id` | parent epic id resolved by `WITH RECURSIVE` walk up `tasks.parent_id` to the nearest type=epic ancestor (NULL if none) |
| `kind` | `'background'` |
| `process` | NULL (bg agents have no tmux pane) |
| `short_id` | `<short_id>` from the dispatch stdout. (Generic column added by this child's schema-change file; harness-agnostic, format determined by `platform`.) |
| `started_at` | now |

This row IS the binding from the endless side. `short_id` is the durable handle: `claude attach <short_id>` works, the row survives the UUID arriving later, queries can locate the bg row by either `short_id` or (eventually) `session_id`.

### SessionStart hook update (bg-context decoration)
The bg agent's `SessionStart` hook fires inside the bg session with `source=startup` and the real `payload.SessionID` (UUID). Hook handler (`internal/hookcmd/claude.go::handleSessionStart`) gets a new branch:

```
if jobDir := os.Getenv("CLAUDE_JOB_DIR"); jobDir != "" {
    short := filepath.Base(jobDir)   // CLAUDE_JOB_DIR ends in /<short_id> per research §6
    // UPDATE sessions SET session_id=<payload.SessionID>, last_activity=now
    // WHERE short_id=<short> AND kind='background'
    // If no row matches → log and fall through to normal new-row path
    // (defensive: the dispatch row should exist, but don't crash if it doesn't).
    return
}
```

The hook skips the worktree-cwd auto-bind path for bg context (bg agents don't have a meaningful cwd for binding). After this UPDATE, the bg row has both `session_id` (real UUID) and `short_id` (dispatch handle), enabling FK links from `session_messages` to actually attach the bg agent's transcript.

### Status of target task
Same gate as `endless task spawn` (fg path): `--force` required for done-ish, `--reopen` reopens terminal. Bg adds nothing here — reuse the existing gate code.

### Throttling
Out of scope for E-1568 — that's **E-1572**. This child unconditionally dispatches.

### Window-name / status-line
No fg tmux window is created for bg agents. The coordinator's window keeps its name. The fg coordinator session's `active_epic_id` is left as-is (the user manages focus separately via `endless task attach` or `endless task spawn --attach`, both delivered by E-1570).

### Schema-change file
E-1571 shipped with `active_epic_id` + `kind` only. This child adds:

- `short_id TEXT` — generic across harnesses (Claude's ~8-hex handle today; future Codex/OpenCode equivalents reuse the same column). The harness discriminator is the existing `sessions.platform` column; format is harness-determined. UNIQUE via partial index (only enforced where NOT NULL).
- `session_id` constraint relaxed: drop `NOT NULL`. Bg rows are inserted with `session_id = NULL` at dispatch; SessionStart UPDATEs to the real UUID when the bg agent's hook fires. UNIQUE constraint preserved.

SQLite has no `ALTER COLUMN DROP NOT NULL`; the change file performs a full table rebuild in a single transaction with `PRAGMA foreign_keys = OFF`:

`internal/schema/changes/e-1568-sessions-short-id-and-nullable-session-id.sql`:

```sql
PRAGMA foreign_keys = OFF;
BEGIN;

CREATE TABLE sessions_new (
    id INTEGER PRIMARY KEY,
    session_id TEXT,                         -- was NOT NULL
    project_id INTEGER,
    platform TEXT NOT NULL DEFAULT 'claude',
    state TEXT NOT NULL DEFAULT 'working',
    active_task_id INTEGER,
    plan_file_path TEXT,
    process TEXT,
    started_at TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%S', 'now')),
    last_activity TEXT,
    transcript_offset INTEGER NOT NULL DEFAULT 0,
    transcript_path TEXT,
    summary TEXT,
    hidden INTEGER NOT NULL DEFAULT 0,
    needs_recap INTEGER NOT NULL DEFAULT 0,
    summary_seq INTEGER NOT NULL DEFAULT 0,
    active_epic_id INTEGER,
    kind TEXT NOT NULL DEFAULT 'tmux',
    short_id TEXT,                           -- new
    UNIQUE (session_id),
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE SET NULL,
    FOREIGN KEY (active_task_id) REFERENCES tasks(id) ON DELETE SET NULL,
    FOREIGN KEY (active_epic_id) REFERENCES tasks(id) ON DELETE SET NULL
);

INSERT INTO sessions_new (id, session_id, project_id, platform, state, active_task_id,
    plan_file_path, process, started_at, last_activity, transcript_offset,
    transcript_path, summary, hidden, needs_recap, summary_seq, active_epic_id, kind)
SELECT id, session_id, project_id, platform, state, active_task_id,
    plan_file_path, process, started_at, last_activity, transcript_offset,
    transcript_path, summary, hidden, needs_recap, summary_seq, active_epic_id, kind
FROM sessions;

DROP TABLE sessions;
ALTER TABLE sessions_new RENAME TO sessions;

CREATE TRIGGER sessions_null_process_on_end_update
AFTER UPDATE OF state ON sessions
WHEN NEW.state = 'ended' AND NEW.process IS NOT NULL
BEGIN
    UPDATE sessions SET process = NULL WHERE id = NEW.id;
END;

CREATE TRIGGER sessions_null_process_on_end_insert
AFTER INSERT ON sessions
WHEN NEW.state = 'ended' AND NEW.process IS NOT NULL
BEGIN
    UPDATE sessions SET process = NULL WHERE id = NEW.id;
END;

CREATE UNIQUE INDEX idx_sessions_short_id_unique
    ON sessions(short_id) WHERE short_id IS NOT NULL;

COMMIT;
PRAGMA foreign_keys = ON;
```

FKs from other tables (`session_messages.session_id`, `session_gates.session_id` referencing `sessions(session_id)`) resolve by name post-RENAME. The `foreign_keys = OFF` block prevents transient violations during the swap.

Mirror the post-migration shape in `internal/schema/schema.sql`: `session_id TEXT` (no NOT NULL) + `UNIQUE`; add `short_id TEXT`; add the partial unique index.

## Files

### New

- `internal/schema/schema.sql` — declare post-migration shape: `session_id TEXT` (drop NOT NULL) + `UNIQUE`; add `short_id TEXT`; add partial unique index `idx_sessions_short_id_unique`. Comment block explaining `short_id`'s harness-agnostic role (paired with the existing `platform` column).
- `src/endless/task_cmd.py`:
  - `spawn_plan(...)` gains `bg: bool = False` and a `--bg` click option declaration.
  - Branch at the post-pre-claim point: if `bg`, skip tmux new-window / send-keys flow, instead build `claude --bg --name f"E-{item_id}"` argv + handoff as positional, `subprocess.run(capture_output=True, cwd=<cd_target>)` where `cd_target` is the worktree (overridden by `--worktree` if supplied).

- `internal/monitor/sessions.go` (whichever file holds inserts) — `RecordBgAgentSession(taskID, projectID int64, activeEpicID *int64, shortID string)` helper (returns the inserted id). Accepts NULL `session_id`.

## Verification

1. `just build && just test` pass.
2. `just land` applies `e-1568-sessions-short-id-and-nullable-session-id` cleanly against a populated DB (test via sandbox: `endless-sandbox init e-test --mode populated`, then `endless db apply-change e-1568-sessions-short-id-and-nullable-session-id`). SQL probes: `SELECT "notnull" FROM pragma_table_info('sessions') WHERE name='session_id';` returns `0`; `short_id` column present; `idx_sessions_short_id_unique` present; both `sessions_null_process_*` triggers present.
3. From a worktree-bound session, claim an epic, then `endless task spawn --bg <child-id>` → command returns near-instant, prints short id, `endless sql "SELECT session_id, kind, active_task_id, active_epic_id, short_id FROM sessions ORDER BY id DESC LIMIT 1" --db main` shows the row with `session_id IS NULL`, `kind='background'`, and `short_id` populated.
4. Wait a few seconds for the bg agent's `SessionStart` to fire; re-query → `session_id` now populated with the real UUID, `short_id` unchanged.
5. `claude agents` (the Claude CLI) lists the bg agent with name `E-<task_id>`.
6. Sanity check: `claude attach <short_id>` from a fresh terminal drops into the bg agent (this dovetails with E-1570).
7. `endless task spawn --bg --worktree /tmp <id>` → bg agent dispatches with cwd=/tmp; row written; no error.
