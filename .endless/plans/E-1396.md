# E-1396 — Add surrogate `id INTEGER PRIMARY KEY` to `session_tasks`

## Context

`session_tasks` shipped in E-1322 (commit `4652b1a`, migrateV9) with `UNIQUE(session_id, task_id)` as its only key and no surrogate `id` column. That violates the project's non-negotiable rule that every table starts with `id INTEGER PRIMARY KEY` as its first column. The rule is recorded in `~/.claude/LESSONS.md` dated 2026-05-16; this task retrofits the one existing table that breaks it and files an Endless decision record so the rule is visible to anyone touching this repo's schema.

SQLite cannot add a `PRIMARY KEY` column via `ALTER TABLE`, so the migration uses the standard table-rebuild dance: create new table, copy rows (ordered by `created_at` so older touches get smaller surrogate ids), drop old, rename new, recreate the index.

### Concurrency safety

A naive rebuild has a race window: a writer that inserts into `session_tasks` between the `INSERT INTO ..._new SELECT FROM session_tasks` and the `DROP TABLE session_tasks` lands in the OLD table and is lost when it's dropped. The framework already anticipates this — `internal/monitor/migrate.go:27-29` defines a `RequiresRebuild` flag for whole-table rebuild migrations; those migrations are refused by auto-migrate and only run when the operator passes `--force-rebuild`. `BackupDB()` runs first (migrate.go:244), so a recovery point exists.

Two-layer safety:

1. **`RequiresRebuild: true`** — moves the timing from "every binary startup, every hook fire" to a deliberate operator action (`endless db migrate --force-rebuild`) that can be scheduled when sessions are quiet.
2. **`BEGIN IMMEDIATE TRANSACTION` wrapping the rebuild** — SQLite acquires a RESERVED lock at BEGIN time, so concurrent writers block until COMMIT. The four rebuild statements (CREATE / INSERT-SELECT / DROP / RENAME) plus the index recreate become atomic. If any step errors, ROLLBACK leaves the original `session_tasks` intact.

### 2. `internal/monitor/db.go` — new `migrateV10`

Place alongside the other `migrateVN` functions (suggest immediately after `migrateV9`). Rebuild dance wrapped in `BEGIN IMMEDIATE TRANSACTION` for atomicity against concurrent writers:

```go
func migrateV10(db *sql.DB) (err error) {
    // session_tasks shipped without a surrogate id (E-1322); SQLite can't add
    // a PRIMARY KEY via ALTER TABLE, so the table must be rebuilt. ORDER BY
    // created_at in the INSERT-SELECT preserves insertion order under the new
    // surrogate ids. BEGIN IMMEDIATE serializes against concurrent writers so
    // a row landing mid-rebuild is not lost when the old table is dropped.
    if !hasTable(db, "session_tasks") {
        return nil
    }

    if _, err = db.Exec("BEGIN IMMEDIATE TRANSACTION"); err != nil {
        return fmt.Errorf("migrateV10: begin: %w", err)
    }
    committed := false
    defer func() {
        if !committed {
            _, rbErr := db.Exec("ROLLBACK")
            if rbErr != nil && err != nil {
                err = fmt.Errorf("%w; rollback also failed: %v", err, rbErr)
            }
        }
    }()

    if _, err = db.Exec(`CREATE TABLE session_tasks_new (
        id         INTEGER PRIMARY KEY,
        session_id INTEGER NOT NULL,
        task_id    INTEGER NOT NULL,
        created_at TEXT    NOT NULL,
        updated_at TEXT    NOT NULL,
        UNIQUE(session_id, task_id)
    )`); err != nil {
        return fmt.Errorf("migrateV10: create session_tasks_new: %w", err)
    }

    if _, err = db.Exec(`INSERT INTO session_tasks_new (session_id, task_id, created_at, updated_at)
        SELECT session_id, task_id, created_at, updated_at FROM session_tasks
        ORDER BY created_at`); err != nil {
        return fmt.Errorf("migrateV10: copy session_tasks rows: %w", err)
    }

    if _, err = db.Exec(`DROP TABLE session_tasks`); err != nil {
        return fmt.Errorf("migrateV10: drop old session_tasks: %w", err)
    }

    if _, err = db.Exec(`ALTER TABLE session_tasks_new RENAME TO session_tasks`); err != nil {
        return fmt.Errorf("migrateV10: rename session_tasks_new: %w", err)
    }

    if _, err = db.Exec(`CREATE INDEX IF NOT EXISTS idx_session_tasks_task ON session_tasks(task_id)`); err != nil {
        return fmt.Errorf("migrateV10: recreate idx_session_tasks_task: %w", err)
    }

    if _, err = db.Exec("COMMIT"); err != nil {
        return fmt.Errorf("migrateV10: commit: %w", err)
    }
    committed = true
    return nil
}
```

Notes:
- All five DDL/DML statements run inside one `BEGIN IMMEDIATE` transaction; if any fail, the deferred rollback leaves `session_tasks` exactly as it was.
- Index recreate is checked too (inside the transaction) — index loss after a successful table rebuild would be a silent perf regression worth surfacing loudly.
- `hasTable` guard handles the no-op case on a fresh DB.
- If `BEGIN IMMEDIATE` fails with `SQLITE_BUSY` (another writer holds a lock), the migration errors and the operator can retry. Acceptable for an operator-triggered command.
