## Changes

### File: `cmd/endless-hook/claude.go`

**Block 1 — direct-commit-to-main blocker (`claude.go:719-731`).** Two stale signals:

- Line 722: `(or 'endless worktree land' once E-971 ships)`. `endless worktree land` shipped with the Layer D mutation slice. Drop the conditional.
- Lines 724/728: example branch name `e-XXX-<slug>`. Wrong convention — Layer F uses `task/NNN-<slug>` for the branch, `e-NNN` for the directory.

Rewrite the help to lead with the supported path (`endless task start`) and keep the manual git fallback only as a secondary option, using the correct conventions:

```
Direct commits to main are not allowed (E-1012).

main is the integration target. Make changes in a worktree on a per-task
branch, then merge via `endless worktree land <task-id>`.

If you have an Endless task for this work:
  endless task start E-NNN          # creates worktree at .endless/worktrees/e-NNN

Or by hand:
  git worktree add -b task/NNN-<slug> .endless/worktrees/e-NNN main
  cd .endless/worktrees/e-NNN
  # ... do work, commit ...
  endless worktree land E-NNN

Bypass (NOT recommended; surfaces as a deliberate violation):
  git commit --no-verify
```

**Block 2 — Edit-in-main blocker (`claude.go:1177-1185`).** One stale signal:

- Line 1183: `via 'endless pivot' (when available)`. `endless pivot` is not a verb in the current CLI. The supported path is `endless task start E-NNN` (which auto-creates the worktree per Layer F). Drop the `endless pivot` reference and recommend the actual flow.

The branch/dir example on line 1184 (`task/NNN-<slug>` / `e-NNN`) is already correct — keep it as the manual fallback.

Rewrite:

```
Edits in main are not allowed (E-971).

main is the integration target — every edit goes through a worktree.

If you do not yet have an active task, create one and start it:
  endless task add "<title>"
  endless task start E-NNN          # auto-creates the worktree

If you already have an active task without a worktree:
  endless task start E-NNN          # idempotent; creates if missing

Or create the worktree by hand:
  git worktree add -b task/NNN-<slug> .endless/worktrees/e-NNN main
```

(The `redirectHint` appended on line 1185 is unchanged — it still produces the "Your active task E-N has a worktree at: <path>" suffix when applicable.)
