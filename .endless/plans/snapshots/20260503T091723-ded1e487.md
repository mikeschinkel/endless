# E-971 Layer F — Auto-create task worktree on `task start`

## Context

Per E-971, every Claude session that edits code lives in a worktree. **Per-session worktrees** (Layer D, shipped in `7199f5f`) auto-create on SessionStart. **Per-task worktrees** persist across sessions and are adoptable; they should auto-create when a plan-bearing task transitions to `in_progress`. Layer F closes that gap so the per-task lifecycle is symmetric with the per-session lifecycle: `endless task start <id>` becomes the moment of birth for a task worktree, mirroring SessionStart for session worktrees.

Today `endless task start <id>` only flips status and emits `task.status_changed`. After Layer F: if the task is plan-bearing (its endless-owned plan file `.endless/plans/E-<id>.md` exists), the command also creates `.endless/worktrees/e-<id>/` on branch `task/<id>-<slug>` from main, writes the companion JSON, and prints a cd-or-spawn hint. Idempotent: if the worktree already exists, it's a no-op with a "already exists" notice.

## Schema reality vs. brief

The brief refers to `tasks.plan_file_path`, but **no such column exists**. Confirmed via `sqlite3 .schema tasks` — the columns are `id, project_id, parent_id, title, description, text, prompt, phase, status, source_file, sort_order, created_at, completed_at, type, updated_at, tier, outcome, focus_id, analysis, notes`. Plan content lives in `tasks.text` and is mirrored to `<project_root>/.endless/plans/E-<id>.md` by `_write_task_plan_file` (`task_cmd.py:278-291`). The mirrored file is the canonical "plan-bearing" signal.

## Decisions

- **Plan-bearing detection**: file existence at `<main_checkout>/.endless/plans/E-<id>.md`. If `tasks.text` is non-empty but the file is missing, treat as not-plan-bearing AND emit a stderr warning — that's a real inconsistency.
- **Worktree directory**: `<project_root>/.endless/worktrees/e-<id>` (lowercase task id only, no slug). Matches recent convention seen in `git worktree list` output (e-967, e-1074, e-1115, e-1135).
- **Branch name**: `task/<id>-<slug>` (no `e-` prefix). Matches convention of recent worktrees.
- **Slug**: per E-971 plan file (line 169) — lowercase title, strip filler words `{a, an, the, to, from, of, for, with, in, on, at, by, and, or}`, replace non-alnum with `-`, collapse repeated `-`, truncate to 40 chars at last `-` boundary. If empty after filtering, fall back to `"task"`.
- **Companion JSON**: `{kind: "task", task_id: "E-<id>", base_branch: <branch>, branch: <branch>, created_at: <ISO UTC>}`. The `kind` field is required for Go-side `WorktreeCompanion` round-trip compatibility (`internal/monitor/worktree_lock.go:34`).
- **Base branch**: detect via `git symbolic-ref refs/remotes/origin/HEAD` and strip prefix; fall back to `"main"`. Keeps the companion truthful for non-main repos.
- **No lock file**: Layer F only writes the companion. The lock is SessionStart's responsibility (Layer D).
- **No `O_EXCL` race protection**: `git worktree add` itself is exclusive on path and branch. The companion write happens after git succeeds, inside a directory only this process owns.
- **No CLI flag to opt out**: Layer D handles the bare-start case if a session lands without a worktree.
- **cwd-agnostic**: create the worktree regardless of current cwd (could be in main, a session worktree, a sibling task worktree). Just print the cd-or-spawn hint; child can't change parent's cwd.

### Step 1: helpers in `src/endless/worktree_cmd.py`

Append after `_normalize_task_id` (line 391). Add `from datetime import datetime, timezone` to the imports.

```python
_FILLER_WORDS = frozenset({
    "a", "an", "the", "to", "from", "of", "for", "with",
    "in", "on", "at", "by", "and", "or",
})


def _slugify_title(title: str) -> str:
    """Slug per E-971 spec for task branch names.

    Lowercase, drop filler words, replace non-alnum with '-', collapse
    repeats, truncate to 40 chars at a word boundary. Returns 'task' if
    the title contains only filler/punctuation.
    """
    cleaned = re.sub(r"[^a-z0-9]+", " ", title.lower())
    words = [w for w in cleaned.split() if w and w not in _FILLER_WORDS]
    slug = "-".join(words)
    if len(slug) > 40:
        truncated = slug[:40]
        if "-" in truncated:
            truncated = truncated.rsplit("-", 1)[0]
        slug = truncated
    return slug or "task"


def _default_base_branch(project_root: Path) -> str:
    """Best-effort detection of the repo's default branch. Falls back to 'main'."""
    try:
        ref = _git(["symbolic-ref", "--quiet", "refs/remotes/origin/HEAD"], cwd=project_root)
    except subprocess.CalledProcessError:
        return "main"
    return ref.removeprefix("refs/remotes/origin/") or "main"


def create_task_worktree(task_id: int, title: str, project_root: Path) -> tuple[Path, bool]:
    """Create the per-task worktree for a plan-bearing task.

    Returns (worktree_path, created) where 'created' is False if the
    worktree already existed (idempotent no-op).
    """
    canonical = f"E-{task_id}"
    slug = _slugify_title(title)
    branch = f"task/{task_id}-{slug}"
    wt_dir = project_root / ".endless" / "worktrees" / f"e-{task_id}"
    base = _default_base_branch(project_root)

    if wt_dir.exists():
        existing = _read_companion(wt_dir)
        if existing and existing.get("task_id") == canonical:
            return wt_dir, False
        raise click.ClickException(
            f"Path {wt_dir} exists but does not belong to {canonical}. "
            f"Resolve manually before retrying."
        )

    wt_dir.parent.mkdir(parents=True, exist_ok=True)
    try:
        _git_run(["worktree", "add", "-b", branch, str(wt_dir), base], cwd=project_root)
    except subprocess.CalledProcessError as e:
        raise click.ClickException(
            f"git worktree add failed for {canonical}:\n{e.stderr or e}"
        )

    companion_dir = wt_dir / ".endless"
    companion_dir.mkdir(parents=True, exist_ok=True)
    companion_data = {
        "kind": "task",
        "task_id": canonical,
        "base_branch": base,
        "branch": branch,
        "created_at": datetime.now(timezone.utc).isoformat(),
    }
    (companion_dir / "worktree.json").write_text(
        json.dumps(companion_data, indent=2) + "\n"
    )
    return wt_dir, True
```

### Integration test (new file `tests/test_task_start_worktree.py`)

Fixture: tmp dir, `git init`, initial commit on `main`, register endless project, insert a task, write `.endless/plans/E-<id>.md`. Then run `start_item`:

1. **Happy path**: plan-bearing task, no existing worktree.
   - `.endless/worktrees/e-<id>/` exists.

6. **`endless worktree list`** afterwards classifies the new worktree `active`; `endless worktree for-task E-<id>` returns the path.
7. **`endless worktree land <id>`** end-to-end on a worktree with one commit succeeds (verifies companion shape).

## Tradeoffs noted

- **`origin/HEAD` may be unset on fresh clones** → `_default_base_branch` falls back to `"main"` even if the actual default is `master` or `develop`. Acceptable for v1; future refinement could read `init.defaultBranch`.
- **Title rename drift**: re-starting a task after a title change retains the original branch slug. This is correct (branches don't auto-rename) but worth knowing.
- **Companion JSON shape change** (added `kind`): existing Python-written companions lack `kind`; Go side already tolerates this since `Kind` is `omitempty` and current Go code only round-trips it. Layer D's `Kind` field is forward-looking, not load-bearing today.
