## CLI surface (final)

```
endless-sandbox run    [--clone] [--name N] [--keep] -- <cmd> [args...]
endless-sandbox enter  [--clone] <name>
endless-sandbox list
endless-sandbox prune  [--older-than DURATION]
endless-sandbox destroy <name>
```

Storage: `~/.cache/endless/sandboxes/<name>/` (XDG_CACHE_HOME-aware).

Meta file at `<sandbox>/.sandbox-meta.json`:
```json
{
  "created_at": "2026-05-01T12:00:00Z",
  "mode": "ephemeral" | "keep",
  "creator_pid": 12345,
  "name": "abc123ef"
}
```

### Behavioral decisions (judgment calls inside the prompt's latitude)

- **`--clone` is deferred.** Flag is accepted by `run` and `enter` but logs `"warning: --clone deep-copy not yet implemented (see E-NEW); sandbox starts empty"` and proceeds. A child task `Implement --clone deep-copy for endless-sandbox` will be filed via `endless task add` on plan approval (parent E-995, references E-1071). Phase 1 does not block on the deep-copy implementation.
- **`enter` always provisions as `mode=keep`.** Entering a sandbox implies a session you might exit and return to; ephemeral semantics don't fit. To dispose, use `destroy <name>` explicitly.
- **`run --name foo` collisions error.** If `foo` already exists, error with `"sandbox 'foo' already exists; use 'endless-sandbox destroy foo' first"`. Force-overwrite is not in scope.
- **List labels:**
  - **Live**: `mode=ephemeral` AND `creator_pid` alive (currently in-flight `run`)
  - **In-use**: `mode=keep` (intentionally persistent, regardless of pid state)
  - **Orphaned**: `mode=ephemeral` AND `creator_pid` dead (E-1068 startup warning will count these)
- **Random ephemeral names:** 8-char hex from `crypto/rand` (e.g. `a7f3c8d1`). Deterministic enough for `list` to display, random enough that two parallel ephemerals never collide.
- **Subshell for `enter`:** `exec.Command(os.Getenv("SHELL"))` with `cmd.Env = append(os.Environ(), "XDG_CONFIG_HOME=<sandbox>", "ENDLESS_SANDBOX=<sandbox>")`. Stdin/stdout/stderr inherited. User runs commands; `exit` returns to parent. Per E-1072.
- **Cleanup ordering:** `run` registers `defer destroy()` and a goroutine listening on a signal channel that calls the same `destroy()` then `os.Exit(130)`. SIGKILL/crashes leave the sandbox on disk by design (E-1068/E-1069 orphan inspection).

## `os.Root` enforcement details

After creating the sandbox dir with regular `os.MkdirAll` (since the dir doesn't exist yet, can't be opened as a Root), call `os.OpenRoot(sandboxDir)`. Pass the `*os.Root` into provisioning code that writes `.sandbox-meta.json` (use `root.Create(".sandbox-meta.json")`) and any clone-data writes (when E-NEW lands). Destroy uses raw `os.RemoveAll(sandboxDir)` since it acts on the sandbox dir itself, not within it.

The `os.Root` test in `sandbox_test.go` will:
1. Provision a tmp sandbox via the same code path
2. Attempt `root.Create("../escape.txt")` → assert error contains "path escapes" or syscall returns EACCES
3. Attempt `root.OpenFile("/etc/passwd", os.O_RDONLY, 0)` → assert error
4. Confirm `root.Create("legit.txt")` succeeds

## Exit codes & output

- `run`: propagates child command's exit code; non-zero on provisioning failure
- `prune`: `0` even if nothing to prune; non-zero on flag errors (e.g. `--older-than 0s` → exit 1, stderr `"minimum --older-than is 24h"`)
- `list`: table to stdout: `NAME  MODE  STATE  AGE  SIZE  CREATOR_PID`. AGE in human-readable (`2h15m`); SIZE via `du`-equivalent walk
- `prune`: stderr scope line first (`"Pruning N orphaned ephemeral sandboxes older than 24h: a b c"`), then on success stdout `"Pruned: a b c"`. No-op message when zero matches.
