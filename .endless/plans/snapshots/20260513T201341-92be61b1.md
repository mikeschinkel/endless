# E-1312 ŌĆö Add `endless session status add` command (XML input)

## Context

Sessions today report status as chat-tables (ephemeral, only Mike-readable). The `session_statuses` table exists in Mike's local DB but isn't in the codebase's schema, and there's no CLI to write to it cleanly ŌĆö current workflow is `endless sql --write "INSERT ..."` per the temp instruction sheet at `docs/private/TEMP-how-to-insert-session-statuses.md`.

E-1312 formalizes this:
- Land the `session_statuses` schema in the codebase so fresh installs have it.
- Add `endless session status add` that takes XML on stdin (or a file path), parses, dedups against the latest row, inserts, and echoes a markdown rendering back to stdout for chat readability.
- Author interface (XML schema) stays stable across v1 (text-blob storage) and v2 (future `session_status_task` child table) per the design discussion.

## Schema

V7 migration in `internal/monitor/migrate.go`:

```sql
CREATE TABLE IF NOT EXISTS session_statuses (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  INTEGER,
    headline    TEXT,
    resolved    TEXT,
    pending     TEXT,
    blocked     TEXT,
    verify      TEXT,
    notes       TEXT,
    created_at  TEXT DEFAULT (strftime('%Y-%m-%dT%H:%M:%S', 'now')),
    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE SET NULL
)
```

Matches the table Mike already created locally (column names + ordering). The migration uses `CREATE TABLE IF NOT EXISTS` so Mike's existing install is a no-op; fresh installs get it created. Bump `CurrentSchemaVersion` from 6 to 7.

## XML schema

Root: `<session-status>`. Children: `<headline>`, `<resolved>`, `<pending>`, `<blocked>`, `<verify>`, `<notes>` (all optional; missing = empty string stored).

Inside `<resolved>`/`<pending>`/`<blocked>`/`<verify>`: zero or more `<task id="E-NNN" status="...">note</task>` elements.

Required attribute validation on `<task>`:
- `id` must match `E-\d+`
- `status` must be one of the known task statuses (`needs_plan`, `ready`, `in_progress`, `verify`, `confirmed`, `assumed`, `completed`, `blocked`, `revisit`, `declined`, `obsolete`)

Parsing: Python stdlib `xml.etree.ElementTree`. Strict ŌĆö unknown root element, missing required attrs, or malformed XML ŌåÆ `click.ClickException` with a clear message.

## Storage

For the four task-section columns, serialize each `<task>` element back to XML and join by newlines (matches the temp instruction-sheet rule "one `<task>` per line"). `headline` and `notes` get the element's text content (or `''` if empty). Missing sections store `''`, never NULL.

## Dedup

Before insert, SELECT the latest row for this `session_id`. If all six text columns (headline, resolved, pending, blocked, verify, notes) byte-equal the new content, skip the insert. Echo a one-line message: "no change since last status; skipped." Otherwise insert.

## Session ID discovery

Same query as the temp instruction sheet:

```sql
SELECT id FROM sessions
WHERE process = ? AND state != 'ended'
ORDER BY last_activity DESC LIMIT 1
```

Bound to `os.environ['TMUX_PANE']`. If `TMUX_PANE` is unset, or the query returns no row, error with a clear "no live session found for pane X" message. A `--session-id N` flag overrides the lookup for testing.

## Input shapes

Two accepted forms:
- **Positional `INPUT_FILE` arg**: `endless session status add path/to/status.xml`. Reads file content.
- **Stdin**: `endless session status add < status.xml` or `endless session status add <<EOF ... EOF`. If no arg given, reads from stdin.

Exactly one source: if both an arg AND stdin are given, prefer the arg (stdin is ignored). Empty input ŌåÆ error.

## Markdown echo

After successful insert (or "no change" skip), render the row to markdown on stdout:

```
## Status
<headline text>

## Resolved
| Task | Status | Note |
|---|---|---|
| E-1208 | confirmed | verbs.jsonl write-time commit |
| ...

## Pending
| Task | Status | Note |
|---|---|---|
| ...

## Blocked
(empty)

## Verify
| Task | Status | Note |
|---|---|---|
| ...

## Notes
<notes text>
```

Sections with no content render as `(empty)` so the structure is visible. The chat sees the same content the row holds.

## CLI registration

```python
@session_cmd.group("status")
def session_status_cmd():

@session_status_cmd.command("add")
@click.argument("input_file", required=False, type=click.Path(exists=True, dir_okay=False))

    """Insert a session status row from XML input."""
```

Implementation lives in a new `src/endless/session_status_cmd.py` module to keep `cli.py` lean.
