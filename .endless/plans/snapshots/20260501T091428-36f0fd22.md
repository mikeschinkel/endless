# E-995: Design sandbox/fixture pattern for tests that mutate state

## Context

Tests and verification work that mutate Endless state — running migrations, adding tasks, emitting events — currently have only one isolation primitive: `tests/conftest.py:isolated_env`, which monkeypatches Python module attributes plus sets `XDG_CONFIG_HOME` and `PATH` for child processes. That primitive works for pytest, but it does not extend to:

- **Binary-level/manual sanity flows** (E-787 Phase 5 hand-rolled `mkdir`/`cp`/`export`/`unset` bash, easy to forget, no programmatic prod-leak guard).
- **Migration spikes against a copy of prod schema** (different from "fresh schema applies cleanly"; see E-1046).
- **Agent workflows that want to "try this thing" without polluting the live DB** (the friction that triggered E-993 → E-995).

The architectural premise is now in place after E-1020: both Python `config.py:8-17` and Go `internal/monitor/db.go:21-28` honor `XDG_CONFIG_HOME`, and pytest's conftest already proves the env-var override works end-to-end across language boundaries. So a "sandbox" is conceptually small — it's an alternate `XDG_CONFIG_HOME` root with its own DB + events + config.json — but it needs ergonomics, safety guarantees, and integration to be a real first-class primitive instead of an ad-hoc bash recipe.

E-995 produces the design + Layer A primitive. Layers C–F can ship as separate child tasks (already filed: E-1043 unified subprocess primitive, E-1044 robust cleanup, E-1045 prod-leak detection, E-1046 migration spike fixture).

## Decisions (recommended; each pulled out for its own E-NNN once confirmed)

1. **Sandbox location: `~/.local/share/endless/sandboxes/<name>/`** containing `endless/` (the XDG-style endless config dir), `metadata.json`, and a `provenance.txt` log. Global rather than per-worktree or per-project: outlives worktree pruning, shares cleanly across sessions, doesn't pollute project `.endless/` trees.
2. **Activation: shell-eval, mirroring `endless session use`.** `endless sandbox use <name>` prints `cd ... && export XDG_CONFIG_HOME=... && export ENDLESS_SANDBOX=<name>`. `endless sandbox exit` prints the corresponding `unset` lines. `endless sandbox run <name> -- <cmd...>` for one-shot use spawns a subshell with the env set + `trap` on EXIT to guarantee cleanup. State file approach rejected — too easy for one shell to silently affect another.
3. **Tripwire: hard-fail.** When `ENDLESS_SANDBOX` is set in the environment, `monitor.ConfigDir()` and Python `config._config_root()` compute the *real* prod path (`<HOME>/.config/endless`, ignoring XDG) and `log.Fatal`/raise if the resolved path equals it. Soft-fail (warn + redirect) explicitly rejected — it's the same shape as the bug E-979 documented.
4. **Pytest dogfoods the primitive.** `tests/conftest.py:isolated_env` is refactored to be a thin wrapper around the new `endless.sandbox.temporary_sandbox()` context manager. Single source of truth for what "isolated" means; ensures the primitive is exercised on every test run.
5. **Sandbox is orthogonal to Endless session.** `ENDLESS_SESSION_ID` and `ENDLESS_SANDBOX` are independent env vars. A session can run inside a sandbox or not; the Go session-tracking code does not need to know about sandboxes. Worth a one-line note in `session use` output: "(active sandbox: foo)" if both are set.
6. **`--copy-from prod` is a first-class flag, not a separate verb.** `endless sandbox create my-spike --copy-from prod` copies both `endless.db` AND the events JSONL log to make rebuild-db reproducible inside the sandbox. `--bare` (default) creates an empty config dir.

## Implementation

### Phase 1 — Core sandbox module + path layout

**New file `src/endless/sandbox.py`** — pure-Python, no Click coupling. Exports:

- `SANDBOX_ROOT = Path.home() / ".local" / "share" / "endless" / "sandboxes"` (overridable via `ENDLESS_SANDBOX_ROOT` for tests of sandbox itself).
- `class Sandbox` — immutable record: `name`, `root: Path`, `endless_dir: Path` (= `root / "endless"`), `metadata: dict`.
- `create(name: str, *, copy_from: Literal["prod"] | None = None, bare: bool = True) -> Sandbox` — creates `<root>/endless/`, runs `db._init_schema` to populate `endless.db`, writes default `config.json`, writes `metadata.json` (created_at, source, schema_version, creator). If `copy_from="prod"`, copies `~/.config/endless/endless.db` and the events JSONL set instead of init-from-scratch.
- `load(name: str) -> Sandbox` — read `metadata.json`, return record. Raises `SandboxNotFound` if absent.
- `list_all() -> list[Sandbox]` — enumerate `SANDBOX_ROOT/*` with metadata.
- `destroy(name: str) -> None` — `shutil.rmtree` after confirming it's a sandbox dir (refuses to delete anything not under SANDBOX_ROOT).
- `activate_env(name: str) -> dict[str, str]` — return the env-var diff to apply (`XDG_CONFIG_HOME`, `ENDLESS_SANDBOX`). Used by `sandbox use`/`run` and by the pytest fixture.
- `temporary_sandbox(*, root: Path | None = None) -> ContextManager[Sandbox]` — context manager: creates a sandbox under `root` (default a tempdir), yields it, destroys on exit. This is what conftest will use.

**Reuse:**
- `db._init_schema` (`src/endless/db.py:38`) for fresh-schema population.
- `db.get_db()` migration path (auto-migrate respects `ENDLESS_AUTO_MIGRATE=1`).
- `internal/monitor/db.go:ConfigDir()` Go-side path resolution (no change needed; honors XDG already).

### Phase 2 — CLI verb wiring

**`src/endless/cli.py`** — new Click group near `worktree_cmd` (around line 1071):

```python
@cli.group(name="sandbox")
def sandbox_cmd():
    """Create and manage isolated sandboxes for tests/verification."""
```

Subcommands:
- `sandbox create <name> [--copy-from prod] [--bare]` → calls `sandbox.create`. Default is `--bare`. Mutually exclusive with `--copy-from`.
- `sandbox list` → table: name, source, created_at, last_used_at, db size.
- `sandbox info <name>` → metadata + paths + currently-active flag.
- `sandbox use <name>` → emits shell to eval. Same shape as `session_use_resolve` (`src/endless/session_cmd.py:1300`):
  ```
  cd <sandbox.endless_dir>  # optional; default: stay in cwd
  export XDG_CONFIG_HOME=<sandbox.root>
  export ENDLESS_SANDBOX=<name>
  ```
  Updates `last_used_at` in metadata.
- `sandbox exit` → emits `unset XDG_CONFIG_HOME ENDLESS_SANDBOX`.
- `sandbox run <name> -- <cmd...>` → fork subprocess with env set, `os.execvpe` (or `subprocess.run`) to the cmd. On Python's exit, env auto-restored (no special trap needed since env applies only to the child).
- `sandbox destroy <name> [--force]` → `sandbox.destroy(name)`. Refuses if `ENDLESS_SANDBOX=<name>` is set in current env (would orphan the active shell).
- `sandbox status` → echo current `ENDLESS_SANDBOX` value + paths if set, else "no active sandbox".

CLI naming: `sandbox` is the noun, all subcommands are verbs operating on it. One positional arg (the sandbox name) per spec.

### Phase 3 — Tripwire (Go + Python)

**Go: `internal/monitor/db.go:ConfigDir()`** — at the top, after computing the resolved path:
```go
if sandbox := os.Getenv("ENDLESS_SANDBOX"); sandbox != "" {
    realProd := filepath.Join(realHomeDir(), ".config", "endless")
    if filepath.Clean(resolved) == filepath.Clean(realProd) {
        log.Fatalf("ENDLESS_SANDBOX=%q set but resolved config dir is the live system path %q — refusing to write", sandbox, realProd)
    }
}
```
`realHomeDir()` reads `os.UserHomeDir()` directly — never via env-var indirection — so a malicious/buggy `HOME` override can't dodge the check.

**Python: `src/endless/config.py:_config_root()`** — same shape, raises `RuntimeError`.

**Coverage check:** every Go binary that opens the DB goes through `monitor.DB()` which calls `ConfigDir()` (`internal/monitor/db.go:36`). Every Python entry that touches config goes through `config.CONFIG_DIR` which derives from `_config_root()`. The tripwire at these two functions covers all known write paths.

**Edge case:** `events.Writer` writes to `<projectRoot>/.endless/events/...`, not `ConfigDir()`. Project-relative writes are out of scope for this tripwire — they're the user's project, not their global config — but document this explicitly so a future mistake doesn't slip through.

### Phase 4 — Pytest dogfooding

**`tests/conftest.py:isolated_env`** — replace the body with:
```python
@pytest.fixture(autouse=True)
def isolated_env(tmp_path, monkeypatch):
    from endless.sandbox import temporary_sandbox
    with temporary_sandbox(root=tmp_path) as sb:
        for k, v in sb.activate_env_dict().items():
            monkeypatch.setenv(k, v)
        monkeypatch.setattr("endless.config.CONFIG_DIR", sb.endless_dir)
        monkeypatch.setattr("endless.config.CONFIG_FILE", sb.endless_dir / "config.json")
        monkeypatch.setattr("endless.config.DB_PATH", sb.endless_dir / "endless.db")
        monkeypatch.setattr("endless.db.DB_PATH", sb.endless_dir / "endless.db")
        monkeypatch.setattr("endless.db._conn", None)
        # PATH override for subprocesses to find local-built binaries
        bin_dir = Path(__file__).resolve().parent.parent / "bin"
        monkeypatch.setenv("PATH", f"{bin_dir}:{os.environ.get('PATH', '')}")
        # Pre-seed default config + projects root
        sb.write_default_config(projects_root=tmp_path / "Projects")
        yield {"config_dir": sb.endless_dir, "projects_root": tmp_path / "Projects", "db_path": sb.endless_dir / "endless.db"}
```
The monkeypatch.setattr lines remain because Python in-process tests bypass the env-var path (modules cache `CONFIG_DIR` at import time). The new piece is that `temporary_sandbox` creates the actual on-disk directory + DB, so the `XDG_CONFIG_HOME` for subprocesses points at a real, populated sandbox — same primitive as binary-level use.

## Out of scope

- Snapshot/restore (sandbox versioning, time-travel).
- Concurrent sandbox access from multiple shells beyond the env-var pattern's natural behavior.
- Web UI for sandbox management.
- Per-project sandboxes — global is enough for now; can layer per-project later if needed.
- `events.Writer` project-local tripwire — events are project-scoped, not config-scoped; out of scope by design.
- Auto-`sandbox exit` shell hook — nice-to-have; ship after the primitive is proven.
- Renaming/cloning existing sandboxes.
