# E-1248: Rename `quick-start` → `guide` and revise the session guide

## Context

`endless quick-start` echoes a markdown file that onboards a Claude Code session to working with Endless. Two problems:

1. **Verb is wrong.** `quick-start` implies a brief intro, but the doc is the canonical session reference. `guide` is honest about what comes out, no semantic mismatch with CLI conventions (unlike `info`, which conventionally means runtime/system state).
2. **Doc is stale.** Written 2026-04-15; today is 2026-05-11. Since then, ~50+ session-facing tasks have shipped and the workflow has changed materially — most importantly `task spawn`/`task prompt`, the per-task worktree model (E-1168), `task start` → `task claim` rename (E-1232), commit-to-main policy (E-1199/1200), and the tmux integration (E-1236). The doc's date-stamped filename also implies a snapshot when the file is meant to be evergreen.

Outcome: the file becomes `docs/GUIDE.md`, the command becomes `endless guide`, and the content reflects what a session needs to know on 2026-05-11.

## New / revised guide content (outline of `docs/GUIDE.md`)

Reorder so the most-needed-by-a-cold-session content is up top:

1. **What Is Endless?** (brief; current text fine)
2. **Status: active development** (current text fine)
3. **You've been given a task — do this** (the happy path):
   - `endless task show <id>` (mention `--text`, `--prompt`, `--children`, `--llm`)
   - `endless task claim <id>` — note this auto-creates a worktree (E-1168)
   - Work in the worktree (cwd matters — see Worktrees section)
   - `endless task update <id> --status verify` when done
   - Report completion with task ID
   - **Do not mark `confirmed`** — only Mike does that
4. **Spawning another session for a task** (new section):
   - Prerequisite: task has `prompt` set (`endless task update <id> --prompt <file>`)
   - `endless task spawn <id>` — opens new tmux window, launches Claude, pastes prompt in /plan mode
   - `--no-plan` to skip plan-mode entry; `--worktree <path>` to override cwd
   - Inside the spawned session, the agent will see the prompt as input — proceed normally
   - The spawned window has tmux window vars `@endless_task_id` and `@endless_project_id`; the session still needs to `task claim <id>` to register ownership (spawn does not auto-claim)
   - `endless task prompt <id>` to print the raw prompt (for piping)
5. **Worktrees (E-1168, E-971)** (new section):
   - Every task gets its own git worktree at `.endless/worktrees/<slug>/`, auto-created on `task claim`
   - Companion metadata at `.endless/worktree.json` records task→worktree mapping
   - `endless worktree current` — what worktree is cwd in
   - `endless worktree for-task <id>` — resolve a task to its worktree
   - `endless worktree list` / `show`
   - **Finishing**: `endless worktree land` — auto-commits endless-managed files, rebases, ff-merges, removes the worktree. Don't merge to main any other way (E-1199).
   - `endless worktree drop` to abandon (refuses dirty/unmerged/foreign without `--force`)
6. **Task lifecycle: statuses, phases, blocking** (consolidate the current three sections; revise status table to list current statuses including `revisit` and clarify `assumed` vs `verify` for blockers)
7. **Field distinctions** (new — single subsection):
   - `title` — one-liner
   - `description` — brief blurb (<200 words), shown in list/show by default (E-1058)
   - `text` — full implementation plan, set with `--text <file>`, shown with `task show --text`
   - `prompt` — task-specific LLM prompt, set with `--prompt <file>`, shown with `task show --prompt`, printable raw with `task prompt <id>`
   - `outcome` — required at terminal statuses (`confirm`, `assume`, `decline`) per E-1021/1022/1175; for decline use `--reason`, for others `--outcome`
8. **Modifying tasks** (CRUD reference — refresh from current `endless task --help`):
   - Existing: add/update/move/remove/search
   - Add to doc: `release`, `decline`, `replace`, `link`/`unlink`/`relations`, `clear`, `import`, `detail`
   - Note `--llm` flag on list/show/next/recent/relations/deps for agent-efficient output
9. **Decisions as first-class items** (new section):
   - `endless decision list` / `endless decision link`
   - Inline rationale on task add/update via `--decision <text>` (E-981) — preferred over manual decision-create
10. **Inter-session channels** (current text fine; light refresh)
11. **Tmux status line** (brief mention):
    - `endless tmux apply` adds a second status row showing `[E-NNNN] · project · status` and a `prefix+e` menu
    - Right-click on status-right also opens menu
    - Useful when running 10+ tmux windows; helps the human see what each session is on
12. **Snapshots** (brief — E-1090):
    - `endless snapshots list/show` to inspect plan-file mirrors written by the PostToolUse hook
13. **Commit-to-main policy** (E-1199, E-1200) (new — critical for sessions):
    - Main's working tree stays clean
    - All work lands via worktree merges (`worktree land`)
    - Exception: global-config artifacts (verbs.json, db-ledger entries) auto-commit to main
14. **Where things live** (refresh from current — keep `db-ledger/` paragraph)
15. **Common patterns** (refresh)
16. **Important notes** (refresh — keep the don't-confirm rule, add: don't bypass `worktree land`, do use `--llm` flag)
