# E-1307 — `endless session id` subcommand

## Context

E-1401 added an attribution gate in `emit_event` that refuses cli/hook events when the resolver can't determine the current session. Its error message already references `ENDLESS_SESSION_ID="$(endless session id)"` as a remediation, but `endless session id` doesn't exist yet — invoking it errors with `No such command 'id'`.

This task ships the command so the recommended form works, then trims the now-redundant "Export ENDLESS_SESSION_ID=<id>" bullet from the gate message (the comment block already at `event_bridge.py:82-84` marks it for removal once this command lands).

## Approach

Reuse the existing 3-layer resolver `_current_endless_session_id` in `task_cmd.py:1971` verbatim. On success, print the integer to stdout. On failure, call `_find_sibling_claude_session` ourselves to produce a layer-specific stderr diagnostic.

### File changes

#### 1. `src/endless/session_cmd.py` — add `session_id_resolve()`

New function after `session_cd_resolve` (around line 1281):

```python
def session_id_resolve() -> None:
    """Resolve the current Endless session id and print the integer.

    For shell substitution: `ENDLESS_SESSION_ID="$(endless session id)"`.
    Success prints just the integer on stdout. Failure writes a one-line
    diagnostic to stderr and exits non-zero. Stdout stays empty on failure
    so command substitution into an env var assigns the empty string
    rather than a diagnostic.
    """
    from endless.task_cmd import (
        _current_endless_session_id,
        _find_sibling_claude_session,
    )
    eid = _current_endless_session_id()
    if eid is not None:
        click.echo(eid)
        return

    # Resolver failed — diagnose which layer for stderr.
    env_id = os.environ.get("ENDLESS_SESSION_ID")
    pane = os.environ.get("TMUX_PANE")
    if env_id and not env_id.isdigit():
        msg = (
            f"ENDLESS_SESSION_ID={env_id!r} is not an integer; "
            "the resolver only accepts digit strings."
        )
    elif not pane:
        msg = (
            "No current Endless session: not in a tmux pane and "
            "ENDLESS_SESSION_ID is unset. Run from a Claude pane or "
            "export ENDLESS_SESSION_ID=<id>."
        )
    else:
        _, n = _find_sibling_claude_session()
        if n == 0:
            msg = (
                "No current Endless session: this pane is not a Claude "
                "pane and no sibling Claude pane was found in this tmux "
                "window."
            )
        else:
            msg = (
                f"Ambiguous Endless session: {n} sibling Claude panes "
                "in this tmux window. Run `endless session id` from the "
                "intended Claude pane, or export ENDLESS_SESSION_ID=<id>."
            )
    click.echo(msg, err=True)
    raise SystemExit(1)
```

Notes:
- `os` is already imported at `session_cmd.py:4`.
- `click` is already imported at `session_cmd.py:7`.
- Imports `_current_endless_session_id` / `_find_sibling_claude_session` lazily inside the function, matching the existing cross-module pattern in this file (see `_current_endless_session_id` itself importing from `session_cmd` at line 1995).
- `raise SystemExit(1)` matches the failure pattern of `session_cd_resolve` (line 1278) — Click's `CliRunner` reports it as `result.exit_code == 1`.

```python
@session_cmd.command("id")
def session_id():
    """Print the current Endless session's integer id to stdout.

    Designed for shell substitution: `ENDLESS_SESSION_ID="$(endless session id)"`.
    Resolves via the same 3-layer logic as cli/hook event attribution:
    ENDLESS_SESSION_ID env var, TMUX_PANE companion match, or a single
    sibling Claude pane in the current tmux window. On ambiguity or no
    match, exits non-zero with a diagnostic on stderr (stdout stays empty).
    """
    from endless.session_cmd import session_id_resolve
    session_id_resolve()
```

#### 4. `tests/test_session_id.py` — new test file

Tests use `@pytest.mark.no_session_stub` to opt out of the conftest autouse fixture that hardcodes the resolver to `lambda: 1`. The CliRunner invocation is `runner.invoke(main, ["session", "id"])`.

Cases:

1. **env var success** — `monkeypatch.setenv("ENDLESS_SESSION_ID", "42")`; assert `result.exit_code == 0` and `result.stdout == "42\n"`.
2. **invalid env var** — set `ENDLESS_SESSION_ID=abc` with no TMUX_PANE; assert non-zero exit, stdout empty, stderr mentions "not an integer".
3. **pane-direct success** — monkeypatch `_current_endless_session_id` to return `7`; assert stdout `"7\n"`, exit 0. (Tests the happy path past env-var without writing companion files.)
4. **no env, no tmux** — `monkeypatch.delenv("ENDLESS_SESSION_ID", raising=False)`, `monkeypatch.delenv("TMUX_PANE", raising=False)`; assert exit non-zero, stderr mentions "not in a tmux pane".
5. **ambiguous siblings** — monkeypatch `_current_endless_session_id` → `None`, `_find_sibling_claude_session` → `(None, 3)`, set `TMUX_PANE=%99`; assert exit non-zero, stderr mentions "Ambiguous" and the count.
6. **zero siblings** — monkeypatch resolver → `None`, sibling finder → `(None, 0)`, set `TMUX_PANE=%99`; assert exit non-zero, stderr mentions "no sibling Claude pane".

Use Click's `CliRunner` with `mix_stderr=False` (or `result.stderr_bytes`) so we can assert stdout and stderr independently.
