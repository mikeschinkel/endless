# E-987 — Resolve event-log auto-append vs clean-main tension

## Context

E-987 is a `research` task whose v1 design (locked 2026-04-30) defined how `endless worktree land` reconciles append-only auto-managed files with the "clean-main precondition" gate. The v1 plan auto-commits dirty endless-managed files on `main` as `Endless: auto-record session activity`, rebases the worktree onto `main`, and ff-merges. For rebase conflicts on auto-managed files, the v1 plan surfaces a copy-pasteable `git checkout main -- <files>` recovery hint.

The 2026-05-11 addendum (commit `36f63c4`) records that E-1240 demonstrated a case the recovery hint mishandles: a worktree branch's legitimate feature commit on `.endless/verbs.json` would be silently destroyed by the suggested `git checkout`. The addendum opened four design questions. This plan resolves them and the broader architectural framing Mike supplied during plan-mode discussion.

## Architectural correction (load-bearing for the resolution)

Mike's framing supersedes the v1 plan's implicit assumption that auto-commit-at-land is the steady-state mechanism:

1. **Main should never be dirty in steady state.** Once endless is hardened, neither endless nor Claude should leave main dirty. Dirt on main is a symptom (errant search-and-replace, agent error, user edit) — not a normal state to absorb.
2. **`land` should NOT auto-commit anything except snapshots.** The current behavior — `land` blindly auto-commits whatever matches `AUTO_COMMIT_GLOBS` — masks bugs and commits files the user never intended. Correct behavior: auto-commit happens at the *point of write* for db-ledger (E-1206) and verbs.jsonl (E-1208); snapshots stay auto-committed at `land` because they're posterity, not transactional.
3. **Auto-managed files are tiered by sacredness, not uniformly prohibited:**
   - `.endless/db-ledger/*.jsonl` — **sacred**. Hand-edits only as a last-resort merge-conflict resolution.
   - `.endless/plans/snapshots/*` — **mildly sacred**. Captured history; hand-edit not a crisis.
   - `.endless/verbs.jsonl` — **not sacred**. Actively maintained; hand-edits on main (if committed) are fine.
4. **The current recovery hint at `worktree_cmd.py:744–759` is actively unsafe** and must be replaced, not deferred. Following it loses changes (the E-1240 case). The replacement is a "investigate this rebase conflict" message — no destructive recipe.

## Resolution of the four addendum questions

### Q1 — Detection of feature-intent vs auto-record on rebase conflict

**Decline the partitioned-detection scope; do the simpler replacement instead.** The partitioned-by-file-class detection with commit-subject probing was the wrong shape: E-1268's `merge=union` driver closed the headline E-1240 case, E-972's routing rule means db-ledger / snapshots can't legitimately conflict from a worktree side, and the broader architectural shift (auto-commit at write time) removes the substrate for these conflicts altogether.

The new recovery hint becomes a single "investigate" message that names the conflicted paths, points at E-972's routing invariant, and refuses to suggest any destructive operation.

### Q2 — db-ledger / snapshot conflict action

**Refuse with "routing bug — investigate."** Per E-972 these shouldn't occur. The replacement hint will name this directly. No auto-discard.

### Q3 — Scope split

**Decline E-1019** (`task update E-1019 --status declined --reason "Partitioned recovery scope replaced by simpler non-destructive 'investigate' hint; bundled with land-auto-commit narrowing under a new task. The original v1+addendum scope is no longer the right shape after E-1268 / E-1206 / E-1208 / E-1216."`). File a fresh task for the bundled change (see "Follow-ups to file" below).

### Q4 — E-1017 hand-edit prohibition

**No textual carve-out for feature commits.** The "hand-edit" wording covers manual edits, which were never going to be the path for legitimate feature work anyway. The corrections Mike supplied are operational tiering, not a wording change.

**Separate concern**: E-1017 still uses the old `events/` directory name. Filed as a standalone follow-up.
