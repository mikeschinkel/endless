# E-1381 — Refine guard fix + verify end-to-end

## Context

E-1322's session_tasks auto-upsert handler shipped with a guard
(`shouldRecordSessionTouch`) that required `Actor.Kind == ActorSession`,
but `Actor.Kind` describes event ORIGIN (cli/hook/web/system), not
authorship. Every real `endless task add` emits `Kind=cli` with
`SessionID` populated from TMUX_PANE resolution, so the guard rejected
100% of legitimate touches and zero `session_tasks` rows were ever
written.

The one-line code fix landed in `cceb60f` on main:
```go
func shouldRecordSessionTouch(evt *Event) bool {
    return evt.Actor.SessionID != ""
}
```
…and that commit also added two new CLI-actor tests
(`TestSessionTasks_CLIActorWithSessionID`,
`TestSessionTasks_NoRowForCLIActorWithoutSessionID`). The fix is real
and effective — ledger entries since `cceb60f` show events arriving
with `"session_id":"451"`.

What's left:

1. **Test realism gap.** The spawn prompt asked that the existing
   positive tests be adjusted to use the realistic production shape
   (`Kind: ActorCLI` with a `SessionID`). `cceb60f` added new CLI
   cases but left the six pre-existing positive tests on
   `Kind: ActorSession` — a shape the production CLI never emits.
   Tests should mirror production so that a future regression in the
   real path (cli + session_id) is caught even if the synthetic
   ActorSession path is left untouched.
2. **Stale `upsertSessionTask` docstring.** Lines 17–20 of
   `session_tasks.go` still claim "the caller's guard already
   restricts this to ActorSession events" — wrong after `cceb60f`.
3. **End-to-end verification not yet run.** Unit tests passing was
   not enough to catch the original bug — the gap that let E-1322
   land broken. The spawn prompt's e2e probe must be executed
   against the real DB to prove the loop closes.
4. **Branch is stale.** This worktree's branch is behind origin/main
   (at least `77c47a5`). User asked me to rebase before verifying.
5. **Task status.** E-1381 is still `in_progress` in the real DB;
   needs to advance to `verify` once the above are done.
