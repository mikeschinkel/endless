E-1125's own description explicitly states it exists "to support a second tmux status line that shows window/task relationships" — but no parent task currently exists for that status line itself. This plan files that parent.

The parent captures the **basic** status line implementation: a runtime command that prints a one-line status string, plus an installer that integrates it into tmux without clobbering an avid tmux user's hand-crafted config. Future feature additions (lineage breadcrumb, sibling summary, focus title, color, format-string config, etc.) become **children** of this parent.

The non-trivial design question this plan resolves: **how does an Endless user install the status line without breaking a power user's `.tmux.conf`?** Recommendation in "Invocation design" below.

## Invocation design — recommendation

Three install paths considered. **Primary: `setup` subcommand. Reject: `shell-init`. Optional sugar: top-level alias.**

| Path | Verdict | Reason |
|---|---|---|
| `endless setup tmux-status` / `setup remove-tmux-status` | **Primary** | Mirrors existing pattern (`setup prompt-hook`, `setup claude-hook`, `setup channel-plugin` — `cli.py:1481–1527`). Users already know `setup` = "make my environment Endless-aware." Symmetric removal verb is free. |
| `endless tmux install-status` (top-level group) | **Optional sugar** | Better discoverability: a tmux user runs `endless tmux --help`. Could be a one-line alias to `setup tmux-status`. Decide at implementation; not load-bearing for this plan. |
| `endless shell-init` extension | **Reject** | Wrong file. `shell-init` (`cli.py:324`) emits bash sourced into `.zshrc`/`.bashrc`. Tmux config lives in `.tmux.conf` with tmux's own syntax. Mixing surfaces confuses both. |

### Don't-clobber strategy (the core decision)

`endless setup tmux-status` is interactive and offers three integration modes; the user picks at install time. **Default never auto-edits `.tmux.conf`.**

1. **Sidecar file + sourced line (default, safest):**
   - Write `~/.config/endless/tmux-status.conf` (lives in the existing global config dir from `internal/config/config.go`).
   - Prompt: *"Add `source-file ~/.config/endless/tmux-status.conf` to your `.tmux.conf`? [y/N/show me the line]"*
   - Re-running `setup tmux-status` regenerates the sidecar; `.tmux.conf` is never re-edited.
   - Avid tmux users get one sourced line they can reorder, comment out, or audit cleanly.

2. **Print-only (`--print`):**
   - Dumps the snippet to stdout. Zero filesystem writes. User integrates manually.
   - This is also the **non-TTY default** — never mutate config silently in scripts.

3. **Direct append (`--append-tmux-conf`, opt-in only):**
   - Writes a fenced block (`# >>> endless tmux-status >>>` / `# <<< endless tmux-status <<<`) to `~/.tmux.conf`.
   - Backs up to `~/.tmux.conf.endless-bak-<ts>` first.
   - `setup remove-tmux-status` strips the fenced block cleanly.
   - For zero-friction installs by users who don't have a custom tmux setup.

The **runtime side** is uncontroversial:
- `endless tmux status` reads `TMUX_PANE` + tmux user vars (`@session_uuid` etc., set by `internal/monitor/tmux.go:GetTmuxContext`), looks up the active session/task/focus, formats a one-line string, prints to stdout.
- Tmux invokes via `set-option -g status-right '#(endless tmux status)'` in the sidecar file.
- Refresh cadence: tmux's normal `status-interval` (default 15s, users often set 5s). Target <100ms per call (one DB read + format).

## Critical files (for the eventual implementation, not this plan PR)

- `src/endless/cli.py` — register `setup tmux-status` / `setup remove-tmux-status` near `cli.py:1481–1527`; register runtime `tmux` command group with `status` subcommand.
- `src/endless/setup.py` — add `install_tmux_status()` / `remove_tmux_status()` mirroring `install_prompt_hook` style.

## Existing utilities to reuse (no duplication)

- `internal/monitor/tmux.go:GetTmuxContext` — tmux session/window/pane UUID resolution; already populates user vars. **Don't re-implement.**
- `src/endless/setup.py` existing handlers (`install_prompt_hook`, `install_claude_hook`, `install_channel_plugin`) — interactive prompts, backup, fenced block patterns. **Mirror exactly.**
- `internal/config/config.go:EndlessConfig` — `~/.config/endless/` is where the sidecar file lives.
- Existing `endless task active` / `endless session show` queries — runtime command should call into shared query helpers, not duplicate DB access.

## What this plan does NOT cover

- The actual code of the status line — lands in a follow-up session, gated on Mike approving the parent task structure.
- E-1125 / E-1128 designs — already their own `needs_plan` tasks.
- A new top-level `endless tmux` command group beyond the runtime `tmux status` verb. If we add `tmux install-status` as sugar, that's a one-line aliasing decision at implementation time.
- Cross-shell concerns — tmux is shell-agnostic; status format is tmux syntax, not bash.
