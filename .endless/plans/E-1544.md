# E-1544 ÔÇö Implement research creation gate with parent-epic exemption and --justification requirement

## 1. Context

ED-1504 established that `--type research` is gated on either (a) parent being an in-progress epic, or (b) the user providing `--justification` explaining why the research can't be inline. This task implements that gate.

Storage of justification text lives in `tasks.notes` under a `## Justification` heading until E-1531 (typed `task_content`) ships; migration to `task_content` is filed under that task, not this one.

Depends on E-1538 (task_types FK seeding with `epic` value) ÔÇö already in flight.

## 2. What to implement

### 2.1 `--justification TEXT` flag

Add `--justification TEXT` to `endless task add` (`src/endless/cli.py`). Repeatable not needed; single string.

### 2.2 Gate logic on `task add`

When `--type research`:

1. If `--parent <id>` is given AND the parent's type is `epic` AND parent's status is `in_progress` Ôćĺ exemption applies, justification not required.
2. Otherwise Ôćĺ `--justification` is required; refuse without it (click.ClickException) and surface the exemption path in the error message.

The parent lookup is a single query: `SELECT t.status, tt.slug AS type_slug FROM tasks t LEFT JOIN task_types tt ON tt.id = t.type_id WHERE t.id = ?`.

### 2.3 Storage

When justification is provided (with or without exemption ÔÇö see Q2 below), write to the new task's `tasks.notes` under a `## Justification` heading. The notes field starts empty for a freshly-added task, so append-after-creation is straightforward.

### 2.4 Tests

- Add a research task with epic-in_progress parent + no justification Ôćĺ succeeds.
- Add a research task with epic-in_progress parent + justification Ôćĺ succeeds and stores.
- Add a research task with non-epic parent + no justification Ôćĺ refuses with exemption-path error.
- Add a research task with epic-but-NOT-in_progress parent + no justification Ôćĺ refuses (Q7).
- Add a research task with no parent + no justification Ôćĺ refuses (Q3).
- Add a research task with no parent + justification Ôćĺ succeeds and stores.

### 2.5 Guide update

`endless guide tasks` should mention the gate in the research-type section.

## 3. Open questions ÔÇö ask Mike before implementing

These are pickup-time decisions. interview Mike interactively and provide context as well as any applicable pros and cons for alternative; don't guess.

**Q1. Gate scope.** Does the gate fire on `task add --type research` only, or also on `task update --type research` (promoting an existing task to research)?

**Q2. Justification when not required.** If `--justification` is passed AND parent is epic+in_progress (exemption applies), accept-and-store it anyway, or reject as redundant? Default-lean: accept-and-store.

**Q3. No-parent research.** If `--parent` is omitted entirely on a `--type research` add, gate fires (no exemption can apply, justification required)? Default-lean: yes.

**Q4. Storage format on `tasks.notes`.** Append a `## Justification` heading + body if absent. What if the user has already written a `## Justification` section by hand? Overwrite, refuse, or append-with-separator? Idempotency matters for `task update --justification`-style re-edits (if Q1 Ôćĺ yes).

**Q5. Error message text.** Needs to surface the exemption (parent must be type=epic, status=in_progress) so the user sees the escape. Draft text for Mike to approve.

**Q6. Migration to `task_content` when E-1531 lands.** Confirm in scope here (no) vs. filed as a separate cleans-up task on E-1531 (yes).

**Q7. Exemption check and sticky-override statuses.** Per E-1537 ┬ž4, `revisit`/`blocked`/`declined`/`obsolete` are sticky-override states that block status auto-derivation. Does any of those satisfy the `in_progress` exemption? Default-lean: only literal `in_progress` exempts; everything else requires justification.

## 4. Verification

1. The five gate-behavior tests in ┬ž2.4 pass.
2. `endless task add --type research --parent <epic-in-progress> "Test"` succeeds without `--justification`.
3. `endless task add --type research "Test"` refuses with a message naming the exemption path.
4. `endless task show <id> --text` (or `--notes` if shown there) displays the stored justification.
5. `endless guide tasks` mentions the gate.
