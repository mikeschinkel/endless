# Agent-facing guide cross-reference + per-command `--help` augmentation

## Context

An agent told to "read the guide and update the spawn command" ran `endless guide`,
saw the index, then ran `--help` on a dozen commands and **never read a guide section** ‚Äî
even though the index already says orchestration covers "spawning sessions."

Two distinct failures:

1. **Reverse-lookup gap.** The index is keyed *by section* ("orchestration covers A, B, C").
   An agent arrives keyed *by command/topic* ("I have `spawn` ‚Äî where do I go?") and must
   read all five prose blurbs and infer the mapping. There is no scannable
   `command/topic ‚Üí section` table.
2. **The `--help` instinct beats the guide.** Even with the blurb present, the agent trusted
   `--help` over the guide. The fix must intercept *at* `--help`: the agent reached for it, so
   put the signpost there.

This change adds (1) a generated `command/topic ‚Üí section` cross-reference in `index.md`, and
(2) per-command `--help` augmentation that, for agents, prepends a directive pointing at the
right guide section. Both are fed by the same per-command map files, regenerated by an
LLM-driven slash command because deciding "which section explains command X" is semantic and
cannot be done well in pure code.

## Converged decisions (settled with Mike)

- **Cross-reference lives in `index.md`** (not a separate section ‚Äî a separate appendix has a
  chicken-and-egg problem). Maps **commands *and* topics**, each to a section + a "covers" phrase.
- **`--help` augmentation:** pointer **prepended above** normal help (normal usage/flags still
  shown below), phrased as a **directive** (agents under-weight soft suggestions). An optional
  short per-command note only where the guide genuinely lacks a command-specific trap. The map
  file *points*; the guide *explains* (no duplication ‚Üí no drift).
- **Agent detection** via the existing `_is_claude_code_agent()` (`CLAUDECODE=1`); a `--agent-view`
  flag forces the agent rendering so a human can see/debug it.
- **Coverage:** every command, top-level and subcommand. A command with no good section is a
  **guide gap, not a map gap** ‚Äî surfaced as a coverage report that drives guide improvement.
- **Maintenance split:** code scaffolds (walk the Click tree, list sections/anchors) ‚Üí LLM fills
  the semantic mapping ‚Üí code validates + assembles. The LLM never enumerates commands from memory.
- **Layering:** deterministic logic in a **Python module** (graduation-ready: this *might* later
  become an end-user `endless` verb, so logic does not live in shell); `just` recipes are dev-side
  wrappers + a drift gate; the LLM step is a **project slash command** (`.claude/commands/`,
  repo-tracked, dev tooling ‚Äî not a shipped verb).

## Components

1. `src/endless/guide_map.py` **(new)** ‚Äî deterministic primitives, importable + testable:
   - `walk_commands(main) -> list[str]`: recursively walk `main.commands` (Click `Group`),
     collecting full command paths (e.g. `task spawn`, `worktree land`). Reuses the existing
     `endless.cli:main` object.
   - `guide_sections() -> dict[slug, list[anchor]]`: glob `docs/guide/*.md` (excluding `index`),
     parse `##`/`###` headers ‚Üí GitHub-style anchor slugs.
   - `load_map(command_path) -> MapEntry | None`: read `docs/guide/help/<kebab-path>.md`.
   - `validate() -> Report`: every command has a map file with a `section:` that resolves to a
     real guide slug; `_topics.md` refs resolve; the `index.md` generated block matches a fresh
     assembly. Returns missing/dangling/stale lists.
   - `assemble_index_block() -> str`: build the `index.md` table from all map files + `_topics.md`.
   - `command_path_to_filename` / back: `task spawn` ‚áÑ `task-spawn.md`.

2. `src/endless/agent_help.py` **(new)** ‚Äî Click help augmentation:
   - Promote `_is_claude_code_agent()` here from `task_cmd.py:226` (re-export there to avoid churn).
   - `_agent_view_requested()`: True if `--agent-view` was seen (set by the root group, below).
   - `class AgentHelpMixin` overriding `format_help(self, ctx, formatter)`: if
     `_is_claude_code_agent() or _agent_view_requested()`, prepend the agent block built from
     `load_map(ctx.command_path-without-"endless")`; then `super().format_help(...)`. Missing map
     file ‚Üí for agents, a generic one-liner ("no section mapped; run `endless guide`"); never errors.
   - `class AgentAwareGroup(AgentHelpMixin, DBAwareGroup)` and
     `class AgentAwareCommand(AgentHelpMixin, click.Command)`, each setting
     `command_class = AgentAwareCommand` / `group_class = AgentAwareGroup` as class attributes so
     the whole tree inherits from one root change.

3. `src/endless/cli.py` **(edits)**:
   - Root group `cls=DBAwareGroup` (line 114) ‚Üí `cls=AgentAwareGroup`.
   - Extend the argv pre-scan in `DBAwareGroup.main()` (lines 82-100) to also consume `--agent-view`
     (strip it, record it via `agent_help`) so Click's per-command `--help` doesn't reject the
     unknown flag ‚Äî identical pattern to `--db`.

4. `docs/guide/help/` **(new dir, generated)** ‚Äî one file per command, dependency-free header:
   ```
   section: orchestration
   covers: Spawning another Claude session ‚Äî the handoff is generated, not authored

   <optional agent note: a command-specific trap not in the guide>
   ```
   - `section:` (required, comma-separated for multi-section) ‚Äî validated against guide slugs.
   - `covers:` (required) ‚Äî short phrase shown in the index table and the `--help` directive.
   - Body (optional) ‚Äî the per-command note.
   - `_topics.md` ‚Äî LLM-curated topic‚Üísection rows (topics aren't enumerable from code), same header
     shape; `_`-prefixed so command-file iteration skips it.

5. `docs/guide/index.md` **(edit)** ‚Äî insert a delimited, regeneration-owned block:
   ```
   <!-- BEGIN generated: command/topic cross-reference (regenerate via /regenerate-guide) -->
   ## Where to look (command / topic ‚Üí section)

   | Command / topic | Section | Covers |
   |---|---|---|
   | `endless task spawn` | orchestration | Spawning another Claude session... |
   | _topic:_ commit-to-main policy | orchestration | ... |
   <!-- END generated -->
   ```
   Only this block is rewritten; hand-authored prose is untouched.

6. `Justfile` **(edits)** ‚Äî thin wrappers over `guide_map.py` (match existing recipe style):
   - `just guide-scaffold` ‚Äî print the command tree + sections/anchors (the skeleton the LLM fills).
   - `just guide-index` ‚Äî reassemble the `index.md` generated block from map files (deterministic).
   - `just guide-check` ‚Äî run `validate()`, non-zero exit on any missing/dangling/stale (pre-land gate).

7. `.claude/commands/regenerate-guide.md` **(new, repo-tracked)** ‚Äî LLM orchestrator:
   run `just guide-scaffold` ‚Üí for each command without/with-stale mapping, read the relevant guide
   section and write/fix `docs/guide/help/<path>.md` (and `_topics.md`) ‚Üí run `just guide-index` ‚Üí
   run `just guide-check` and report remaining coverage gaps as guide-improvement candidates.

## Agent `--help` rendering (example: `endless task spawn --help`)

```
‚ñ∏ AGENT ‚Äî read this before using this command:
    endless guide orchestration   (Spawning another Claude session ‚Äî the handoff is generated, not authored)
    <optional one-line note>

Usage: endless task spawn [OPTIONS] ...     <- normal Click help unchanged, below
...
```

Humans (no `CLAUDECODE`, no `--agent-view`) see only the normal Click help.

## Verification

1. `just build` (or editable install already live). Run from the worktree.
2. Coverage gate: `just guide-check` exits non-zero and lists any unmapped command ‚Äî add a map
   file for each until it passes. Then `just guide-check` is green.
3. Agent view: `CLAUDECODE=1 endless task spawn --help` shows the directive block above usage;
   `endless task spawn --help --agent-view` (no `CLAUDECODE`) shows the same; plain
   `endless task spawn --help` shows only normal help.
4. Drift gate: rename/add a `##` header in a guide section, run `just guide-check` ‚Üí it flags the
   dangling/stale ref. Add a new subcommand stub ‚Üí `just guide-check` flags the missing map file.
5. Index: `endless guide` prints the cross-reference table; `endless task spawn` and at least one
   topic row resolve to the right section.
6. `just guide-index` is idempotent (running twice produces no diff).
7. Tests: add `guide_map.py` unit tests (tree walk count, anchor slugging, validate() on a fixture
   with a missing file and a dangling ref). `just test`.
