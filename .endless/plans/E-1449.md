# Delete the plan-snapshot feature

Implements E-1448's decision to remove plan snapshots entirely. Snapshots were a
forensic recovery tool (E-969, hardened by E-1275): every Write to
`~/.claude/plans/*` was captured to
`<project>/.endless/plans/snapshots/<ts>-<sha8>.{md,json}` and committed at write
time. The recovery scenario never materialized in ~3 weeks of production while
the feature accreted follow-on complexity. Reversible by reverting the deletion
commit.

## Decisions (confirmed with Mike, 2026-05-21)

1. **`AMENDABLE_COMMIT_SUBJECTS` `'Endless: snapshot plan'`** → removed (full
   delete), with the two land tests updated accordingly.
2. **`src/endless/plan_cmd.py`** → deleted (the file became empty once the
   snapshot functions were removed; its only importers were the deleted CLI
   commands).
3. **Historical plan docs** → **left as-is.** The grep surfaced 11
   `.endless/plans/E-*.md` files referencing snapshots; several use them as
   load-bearing context for *surviving* features (E-1342, E-987, E-1309,
   E-1113), and these files are committed mirrors of the DB's `tasks.text` (the
   real source of truth). Only the live user-facing doc (`docs/guide/reference.md`)
   was scrubbed.

## What was deleted

**Go**
- `cmd/endless-hook/claude.go`: `snapshotPlanFile`, `existingSnapshot`, the
  PostToolUse call site + comment block, and the now-unused imports
  (`crypto/sha256`, `encoding/hex`, `internal/events`). The `isPlanFile` guard
  and `monitor.SetPlanFilePath` call were **kept** (ExitPlanMode tracking).
- `internal/events/commit.go`: `CommitSnapshotPair`, the `SnapshotCommitSubject`
  constant, and snapshot mentions in the package/`commitPaths` docs. All shared
  helpers and `CommitLedgerSegment`/`LedgerCommitSubject` kept.
- `internal/events/commit_test.go`: the 5 `TestCommitSnapshotPair_*` tests and
  their two snapshot-only helpers (`initRepoForSnapshot`, `addSnapshot`). The 9
  ledger/git-env tests kept.
- `internal/monitor/db.go`: `IsSandboxActive` (sole caller was
  `snapshotPlanFile`). `ProjectPath` kept (7 other callers).
- `internal/monitor/sandbox_test.go`: deleted entirely.

**Python**
- `src/endless/plan_cmd.py`: deleted.
- `src/endless/cli.py`: the `snapshots` command group + `list`/`show`
  subcommands. The unrelated `session status` records (E-1312) kept.
- `src/endless/worktree_cmd.py`: removed `.endless/plans/snapshots/*` from
  `AUTO_COMMIT_GLOBS` and `'Endless: snapshot plan'` from
  `AMENDABLE_COMMIT_SUBJECTS`; scrubbed three incidental comment/docstring
  mentions.
- `tests/test_worktree_land_orphan_drop.py` and
  `tests/test_worktree_land_dirt_guard.py`: recast the snapshot-based cases to
  use a second `db-ledger` segment (the helpers' contracts are about branch
  shape / auto-managed-glob membership, not the snapshot type specifically).

**Filesystem & docs**
- `git rm -r .endless/plans/snapshots/` → 116 files (58 `.md`/`.json` pairs).
  One stray untracked pair (sandbox-mode write from this session's plan-file
  edit under the old hook) was also removed.
- `docs/guide/reference.md`: removed the "Plan snapshots" section, the
  file-layout table row, and the "snapshot auto-commits" mention.

## Verification (all pass)

1. `just build` — succeeds, no references to deleted symbols.
2. `just test` — 414 passed (incl. the two rewritten land tests).
3. `go test ./...` — exit 0.
4. ExitPlanMode plan-file tracking preserved: `isPlanFile` →
   `SetPlanFilePath` → `GetPlanFilePath` chain intact (verified at code level;
   exercised naturally in any plan-mode session).
5. `endless snapshots --help` → "No such command 'snapshots'."
6. `git status` clean of snapshot artifacts (only this task's `E-1449.md`
   remains untracked, then committed).
7. Rebuilt `bin/endless-hook` contains no snapshot path code (no `snapshots`
   strings; PostToolUse handler has no snapshot branch).
