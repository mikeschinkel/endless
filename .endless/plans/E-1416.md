# Pre-rebase check in `worktree land`: verify the dirty-tree guard

## Context

`endless worktree land` invokes `git rebase` without a structured pre-flight
check of the worktree's working tree. When the tree is dirty, rebase fails
partway with a generic "You have unstaged changes" error — no file list,
no recovery hint. E-1416 ("Eliminate uncommitted ledger artifacts as a
worktree-land failure mode") commits to fixing this.

The originating failure mode was specifically `.endless/` ledger /
snapshot artifacts left untracked by `task update --status verify`. The
structural fix is being delivered by **E-1361 / E-1362 / E-1363 / E-1364**
(moving plan snapshots and verbs.json out of the working tree). E-1416 is
the safety net that catches whatever the structural fix misses — a future
writer added without auto-commit hygiene, a manual user edit, a
partially-staged plan file.

## Current state

Commit `3e22fd1` ("E-1416: guard worktree land against dirty worktree
tree before rebase") already added the guard. The flow is now:

- `_guard_dirty_worktree(worktree_path, branch, canonical)` at
  `src/endless/worktree_cmd.py:572-614`
- Invoked at Step 3.8 (`src/endless/worktree_cmd.py:1020`), immediately
  before the `git rebase` at Step 4 (line 1024).
- Reuses `_git_status_partition()` (line 522) and the
  `AUTO_COMMIT_GLOBS` set (line 54) — keeping classification logic
  consistent with the existing Step 1/2/3 pattern on main.

The guard has two refusal branches:

1. **Auto-managed dirt** (path matches `AUTO_COMMIT_GLOBS`):
   error message identifies the file(s) and frames the cause as
   "writer is broken or skipped its commit. Report the writer; do
   not auto-commit manually."
2. **Everything else** (any other dirty path):
   error message lists files and offers three recovery routes —
   commit on the branch, move the file aside, or revert.

## Decisions (confirmed)

- **Scope = broad**: keep aborting on any dirty path in the worktree,
  not just `.endless/`-prefixed paths. No separate task addresses
  outside-`.endless/` dirt as a land failure mode (verified by ledger
  search), so narrowing would leave a gap. Spec text reading "rebase
  surfaces its own message" for outside-`.endless/` dirt is rejected —
  rebase's native error is unhelpful for ANY dirt class, and the broad
  guard's structured message is strictly better UX.
- **Message wording = unchanged**: keep the bulleted
  "commit / move aside / revert" recovery options in the
  unmanaged-dirt branch. A literal `git add <files> && git commit -m
  "Endless: record ledger entry"` example is deferred — picking the
  right commit subject depends on the file's role (ledger vs.
  snapshot vs. manual edit), and a wrong example would mislead. File
  a follow-up to refine the message once we see a concrete failure
  scenario that justifies a specific recipe.
- **Two-branch refusal = preserved**: auto-managed dirt and user
  dirt continue to produce distinct error messages. The auto-managed
  branch is informative because it signals an upstream writer bug; the
  user-dirt branch covers manual edits and offers generic recovery.

## Work to do

The implementation is already in place, so the remaining work is
verification and (optionally) regression coverage.

### 1. Manual verification

From a worktree spawned for this branch (or a fresh scratch worktree
created for the purpose — do NOT verify in `e-1416` itself since land
removes the worktree on success):

1. **Auto-managed dirt — untracked ledger file**
   ```sh
   touch .endless/db-ledger/test.jsonl
   endless worktree land <task-id>
   ```
   Expected: abort BEFORE rebase, message names the file and
   contains "writer is broken or skipped its commit". Verify rebase
   was not invoked (e.g., no `REBASE_HEAD` reflog entry).

2. **Auto-managed dirt — untracked snapshot file**
   ```sh
   touch .endless/plans/snapshots/test.md
   endless worktree land <task-id>
   ```
   Same expected behavior as (1).

3. **Unmanaged dirt under `.endless/`**
   ```sh
   touch .endless/scratch.txt
   endless worktree land <task-id>
   ```
   Expected: abort BEFORE rebase, message names the file and offers
   the three bulleted recovery options.

4. **Unmanaged dirt outside `.endless/`**
   ```sh
   touch src/scratch.txt
   endless worktree land <task-id>
   ```
   Expected: abort BEFORE rebase with the same user-dirt message
   (confirms broad scope).

5. **Modified-but-unstaged file**
   ```sh
   echo X >> .endless/worktree.json
   endless worktree land <task-id>
   ```
   Expected: abort with `.endless/worktree.json` named in the
   unmanaged-dirt branch's message.

6. **Clean worktree** — control: land succeeds end-to-end.
