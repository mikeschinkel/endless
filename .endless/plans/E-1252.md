# E-1252 — Rework task-status vocabulary — PLAN

Decided 2026-06-24 (see ED-1513). Implements the rename half of the status-model
rework; the values table + FK + `blocked` drop are E-1532, the approval-gate
behavior is E-1648.

## Decided vocabulary
| old | new | note |
|---|---|---|
| `needs_plan` | **`unplanned`** | no plan yet; no underscore |
| (new) | **`unapproved`** | plan attached, awaiting Mike's approval (the gate's review state) |
| `in_progress` | **`underway`** | |
| `verify` | **`unverified`** | done, awaiting verification |
| `blocked` | — | dropped (E-1532); blocking is the `blocked_by` relation |
Unchanged: `ready` (now = approved), `confirmed`, `assumed`, `completed`,
`revisit`, `declined`, `obsolete`. The three "un-" states (`unplanned` /
`unapproved` / `unverified`) are the gate-awaiting states.

## Implementation
The data + seed changes ride **E-1532's `.go` change-file** (one rebuild):
`UPDATE tasks SET status=<new>` per renamed slug; seed `task_statuses` with the
new slugs. This task owns the **reference sweep** off the old strings.

## Reference sweep (every occurrence of `needs_plan`/`in_progress`/`verify`)
- **Python**: `cli.py` `TASK_STATUSES` + `click.Choice`/help text; `task_cmd.py`
  status literals and logic (`== 'needs_plan'`, `== 'in_progress'`, `== 'verify'`,
  the `valid` tuple, `task active`/`next`/`show` filters); any other module.
- **Go**: `internal/monitor`, `internal/events`, `internal/tmuxcmd`, web — any
  status string literals.
- **Docs (Mike's call-out)**: `endless guide` (status table, happy path, blocking
  semantics), **CLAUDE.md** (project + global — they reference statuses and the
  "set verify when done" step), **README.md**, **handoff templates**
  (`docs/templates/handoff*` + per-type variants), `agent_help` / guide
  cross-reference.
- **Web UI**: status labels/badges.
- Re-grep at implementation time; the list above is the known surface, not a
  guarantee of completeness.

## Note
Workflow-meaning changes (`ready` = approved, the `approve` step, `unverified`
phrasing in the verify-handoff) are **E-1648's** doc edits — coordinate so the two
sweeps don't clobber the same handoff/guide sections. This task changes the
*names*; E-1648 changes the *workflow prose*.
