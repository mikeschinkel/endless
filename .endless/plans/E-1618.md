# Plan — verify.toml [[check]] list (A+B hybrid)

Revises E-1602's landed single-`runner` manifest so one verification composes multiple runners.
Shared design is in the epic's analysis (`endless task show E-1596 --analysis`, Interface
Contract / A). Conform to it.

## The model

A verification is a list of `[[check]]` entries; each is one runner invocation contributing to
the ticket's proof. Two forms:

- **First-class runner (B):** `runner` is a known runner (`gotest`, `pytest` to start) + a
  structured `tests` list of runner-native identifiers. Endless translates to the native filter
  (`gotest` → `go test -run 'TestA|TestB' ./...`; `pytest` → `pytest <nodeids>`). `format` is
  inferred from the runner.
- **Raw command (A, fallback):** any other runner uses `command` (the literal invocation) +
  declared `format` (default `tap`). This is the escape hatch for runners Endless doesn't
  translate.

```toml
schema = 1
task   = "E-1234"

[[check]]
runner = "gotest"
tests  = ["TestFoo", "TestBar"]

[[check]]
runner = "pytest"
tests  = ["tests/test_x.py::test_a"]

[[check]]
runner  = "bats"
command = "bats ./.endless/tasks/E-1234/cli.bats"
format  = "tap"
```

## Deliver

1. Replace the single `runner`/`format` with the `[[check]]` array in the Go manifest struct +
   parser (extends E-1602). `setup`/`seed`/`needs`/`tiers` stay top-level.
2. A **first-class runner registry**: `gotest`, `pytest` → (a) `tests`→native-filter translation,
   (b) format inference. Adding a runner later = add one entry.
3. **Validation:** `tests` legal only on a first-class runner; otherwise require `command`.
   Unknown first-class runner with `tests` fails loudly.
4. **Bare-clone emit:** Endless emits the resolved native commands (a generated `run.sh` /
   README one-liner) so a no-Endless clone runs every check on exit code alone. `command` checks
   are already concrete.

## Notes

- `tests` entries are runner-native identifiers (Go test names, pytest nodeids) — structured in
  that Endless knows the runner and that they are selectors, not cross-runner-uniform IDs.
- Execution + merge live in the runner (E-1603) and normalizer (E-1604): run all checks,
  normalize each by its `format`, merge into one CTRF report.

## Verify

- A manifest with a `gotest` + a `pytest` + a `bats` check parses; `tests` on `bats` fails
  validation; `command` missing on a non-first-class runner fails.
- `gotest`/`pytest` `tests` lists translate to the expected native filter commands.
- The emitted bare-clone command set runs every check with no Endless present.
