# E-1186: Update E-971 plan — remove "no create verb" lock and align dir convention

Parent / Documents: E-971.

## What's wrong with E-971's current plan

Two items in `.endless/plans/E-971.md` need correction:

1. **"Decisions locked" → "No `create` verb."** Derived from a soft preference ("worktree creation should ideally be a side-effect of session/task start") and transcribed as a hard prohibition. The transcription was a misreading; the original preference left room for cases the auto-triggers don't cover.

2. **Dir convention `.endless/worktrees/<id>-<slug>`.** Does not match what's documented in CLAUDE.md (`e-NNN`), in E-1150's description (`e-NNN`), or what's now on disk (the dirs were renamed to `e-NNN` form during this conversation). E-971's plan is the outlier.

The verify-worktree workflow — creating a fresh worktree to manually re-test a completed/merged task — does not fit either of E-971's auto-creation triggers. It surfaced through dogfooding and required manual `git worktree add` + companion-file edits, proving the gap.

## Edits to `.endless/plans/E-971.md`

### Section "Decisions locked"

**Remove**: "No `create` verb."

**Do not add a softer replacement.** Considered "Worktree creation usually happens as a side-effect; explicit verbs cover the gaps" — rejected as unnecessary. The verbs document themselves in their own sections; an overarching meta-rule is just ceremony.

**Keep**: "No `switch` verb." That one is a real technical constraint (a child process can't change its parent shell's cwd), not a misderived preference.

### Section "Layout inside the repo"

Replace `.endless/worktrees/<id>-<slug>` with:

- `.endless/worktrees/e-NNN` (canonical)
- `.endless/worktrees/e-NNN-<differentiator>` (variants — e.g., `e-1130-verify`)

### Section "Worktree creation triggers"

Add a third subsection after **Per-session worktree** and **Task worktree**:

> **Explicit (`endless worktree add`).** For cases neither auto-trigger covers — primarily verify worktrees for manual re-testing of completed work. See E-1190.

In the existing subsections, clarify branch convention: `task/NNN-<slug>` for canonical task worktrees, `task/NNN-<differentiator>` for variants. Slug remains in the *branch* name; only the *dir* name drops it.

## Decision capture

Create a separate decision entity via `endless task add --type decision`:

- **Title**: Removed "No create verb" lock from E-971; verify use case discovered via dogfooding
- **Body**: The original lock was misderived from a soft preference. The verify-worktree workflow does not fit either of E-971's auto-creation triggers. An explicit `worktree add` verb covers the gap without contradicting the design's intent (most worktree creation still happens as a side-effect; the explicit verb is the escape hatch).
- **Link**: `--documents E-971`
