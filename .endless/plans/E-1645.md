## Behavior changes
### 1. Liveness guard â†’ navigate, don't just error
Today `_check_task_ownership` raises when a different live session owns the task. For reopen,
branch on the live session's kind:
- **Foreground (tmux, has `process` pane):** `tmux switch-client -t <pane>` (**`switch-client`,
  not `select-window`** â€” works across sessions). Exit **without spawning**.
- **Background (`kind_id=2`):** print `attach with: endless task attach E-<id>` and exit.
- **No live session:** proceed. `--new-session` does NOT bypass a live owner.

### 2. Ensure / rebuild worktree + sandbox
- Present (not reaped): reuse (`worktree_cmd.py:1071-1080`).
- Reaped by land/drop: fresh branch **off `main`**. Collision policy: reuse `task/<id>-<slug>` if
  it still exists (drop case); a land-deleted branch means a fresh branch off main already holds
  the work. Decide + document the suffix rule.
- Sandbox: if reaped, re-`Provision` + re-`bind` (bind requires an existing sandbox â€” init first).

### 3. Inherit the most-applicable prior session (default)
Live-first is handled by the guard; among *ended* rows:
```sql
SELECT id, session_id, process, transcript_path, started_at, last_activity,
       (julianday(last_activity)-julianday(started_at))*86400 AS dur
FROM sessions WHERE active_task_id = :task AND state='ended'
ORDER BY (process IS NOT NULL OR transcript_path IS NOT NULL OR dur >= 10) DESC, started_at DESC
LIMIT 1;
```
- **Skip sub-10s ghosts; never order by `dur` desc** (Pattern B makes a stale row span days).
  Defends against E-1640 ghosts regardless of whether E-1640 has landed.
- Use the inherited session's worktree (maybe an alt) + its `outcome` + latest `session_statuses`
  to populate the E-1647 handoff vars. `--new-session` â†’ fresh row + fresh worktree off main.

### 4. Render the flat respawn handoff
`render_handoff(..., task_type="respawn", restore_case=â€¦, prior_outcome=â€¦, last_status_snapshot=â€¦)`.
Keep fg + `--bg` parity.
