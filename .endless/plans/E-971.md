# W3 — Worktrees and the Behavioral Gate

> Part of E-968 (parallel-session support). See `.endless/plans/E-968.md` for full context. Blocked by E-969 (W1) and E-970 (W2).

## Goal

Two complementary mechanisms, same goal: never let work happen without an explicit task association.

## Decisions locked

- **No h2 / h2pp absorption.**
- **No DB worktrees table.** Filesystem and git are authoritative.
- **Layout inside the repo**: `.endless/worktrees/<name>/`, gitignored. Aligns with JetBrains' worktree support (location-agnostic via `git worktree list`).
- **Configurable path.** Endless must co-exist with other worktree managers, not own.
- **Not per-task by default.** Hybrid: only **plan-bearing** tasks earn a worktree.
- **Worktrees are agent-facing, not user-facing.** The `endless worktree` verb is primarily an API for Claude.
- **No `create` verb.** Worktree creation is always a side-effect of starting a plan-bearing task.
- **No `switch` verb.** A child process can't change its parent shell's cwd; switching means a new session.
- **Spawn stays narrow.** Don't couple worktree creation to `task spawn`.
- **Overlap detection deferred.** Mike coordinates manually short-term.

## Worktree trigger: plan-bearing task transitions to `in_progress`

A task is **plan-bearing** if `tasks.plan_file_path` is set (and points at the endless-owned file from W1, E-969). That's the entire definition. No `--isolated` flag, no children-detection.

When `endless task start <id>` runs on a plan-bearing task:

1. Compute branch name `task/<id>-<slug>`.
2. If a worktree for that branch exists, do nothing.
3. Otherwise: `git worktree add -b <branch> .endless/worktrees/<id>-<slug> <base>`, write `.endless/worktree.json` sidecar `{task_id, base_branch, branch, created_at}`.
4. Print a cd-or-spawn hint if the current session's cwd is elsewhere. Hook handles enforcement.

## Worktree lifecycle (derived state, no DB)

| State | Derived from |
|---|---|
| **active** | `git worktree list --porcelain` shows it AND `.endless/worktree.json` present |
| **merged** | branch in `git branch --merged <base>` |
| **abandoned** | unmerged, no commits in N days, no live session bound; surfaced not auto-cleaned |
| **foreign** | listed by git but no endless sidecar; left alone |

Endless never deletes a worktree it didn't create.

## Behavioral gate (UserPromptSubmit hook)

Two layers, no semantic guessing.

**Layer 1 — always-on context refresh.** Inject one line into Claude's context as a system reminder: *"Active task: E-XXX — <title>."* No comparison, no decision required. Keeps the active task salient at the moment context is most likely to drift.

**Layer 2 — deterministic gate.** Match the user's message against `kind='pivot'` rows in the phrases table (E-970). Match → block Write/Edit until Claude clears the gate via CLI verb (see clearance rules below). Same gate, same clearance, regardless of whether the trigger was case-sensitive ("PIVOT") or case-insensitive ("actually"); the case-sensitivity flag distinguishes deliberate from incidental typing, not severity.

Worktree-bound work gets the gate automatically: the worktree IS the scope declaration, so PreToolUse already enforces task↔worktree match. Main-checkout work uses Layer 1 + Layer 2.

**Gate clearance.** Two paths, friction proportional to risk:

- **Continuation** (`endless task start <existing-id>` or `endless task confirm <id>`): Claude self-clears. No user approval needed; the user just typed the trigger and is in the loop. If Claude declares wrong, the user corrects on the next turn.
- **New task creation** (`endless task add ...` triggered as gate clearance): brief draft-and-approve flow. Claude proposes a title and parent based on the user's message; user accepts or edits. One content-bearing turn.

Avoids approval fatigue while preserving oversight where it matters.

## Hook integration summary

- **SessionStart**: read cwd, walk up for `.endless/worktree.json`, bind worktree identity in-memory.
- **UserPromptSubmit (new event)**: inject Layer 1 reminder; check phrases table for Layer 2 triggers.
- **PreToolUse**: existing task-active check + worktree↔task match for plan-bearing tasks.
- **Stop / SessionEnd**: nudge to merge if task moved to `verify` and worktree clean.

## CLI surface (Claude is the primary consumer)

Output is `--json` capable, exit codes stable, no interactive prompts.

**Introspection (high frequency):**
- `endless worktree current` — what worktree am I in? bound task? dirty/clean/ahead/behind?
- `endless worktree list [--status active|merged|abandoned|foreign]`
- `endless worktree show <name>`
- `endless worktree for-task <task-id>` — resolve task to worktree path or "not isolated".

**Mutation:**
- `endless worktree land <task-id>` — implements the safe local-only landing dance (see Land algorithm below). Refuses on `foreign`. `--pr` flag uses remote-based path instead when a remote is configured.
- `endless worktree drop <name>` — explicit cleanup; refuses unmerged/dirty without `--force`. Refuses on `foreign`.

## Land algorithm

**Goal.** Ship the safety dance as the easy path. No stash (unsafe; can drop untracked files, half-apply on conflict, requires reflog spelunking to recover). Local-only by default; PR-based remains an option when a remote is configured.

**Why this matters.** Mike's main checkout will commonly have unrelated WIP from other sessions. `git merge --ff-only` fails against dirty main. This is the common case, not the exception, so the CLI must handle it.

**Steps (local-only, default):**
1. Resolve the worktree and branch for `<task-id>`.
2. Switch to the base branch's checkout (the main repo).
3. If base has uncommitted changes: `git commit -am "WIP: parked for landing <task-id>"`. This creates a real, reflog-recoverable commit holding the WIP.
4. `git merge --ff-only <branch>` to land the worktree's commits.
5. If step 3 ran: `git reset --soft HEAD~1` to restore the parked WIP to the index, leaving the landed commits intact above it.
6. `git worktree remove <worktree-path>`.

**Recovery.** Any mid-dance failure is restorable via `git reset --hard ORIG_HEAD`. The worktree itself still exists on disk until step 6, so retry is non-destructive. Stash is **never** used.

**PR-based path** (`--pr` flag, requires remote): push branch, `gh pr create`. Worktree stays until the PR merges and the user runs `endless worktree drop`.

**Acceptance criteria:**
- (a) `endless worktree land <task-id>` succeeds against dirty base without losing WIP.
- (b) Any failure is restorable via `git reset --hard ORIG_HEAD`.
- (c) `git stash` is never invoked anywhere in the path.

## Co-existence with other worktree managers

- `worktree_dir` configurable in `.endless/config.json` (default `.endless/worktrees`).
- Discovery via `git worktree list --porcelain` + sidecar filter, not path-prefix.
- `merge` / `drop` refuse on worktrees lacking an endless sidecar.

## Sidecar format (JSON)

```json
{
  "task_id": "E-808",
  "base_branch": "main",
  "branch": "task/808-event-logs-authoritative",
  "created_at": "2026-04-28T15:30:00"
}
```

## Slug generation

Lowercase, strip filler words (`a`, `an`, `the`, `to`, `from`, `of`, `for`, `with`, `in`, `on`, `at`, `by`, `and`, `or`), replace non-alnum with dash, collapse repeated dashes, truncate to 40 chars at a word boundary.

Example: task E-757 "Move title verbs from hardcoded list to database table" → slug `move-title-verbs-hardcoded-list-database`. Branch: `task/757-move-title-verbs-hardcoded-list-database`.

## Verification

- Two parallel sessions on two plan-bearing tasks; each gets its own worktree; no edit collisions.
- Plan-bearing task active, cwd is main checkout; PreToolUse blocks with switch-worktree message.
- User types "actually let's fix the auth bug"; gate fires; Claude can't edit until task declared.
- User types "PIVOT" mid-message; gate fires regardless of context.
- Small ad-hoc task `in_progress`; no worktree created; existing session keeps editing freely.
- Manually `git worktree add` outside endless; `endless worktree list` shows it as `foreign`; land/drop refuse.
- `endless worktree land <task-id>` against a base with 14 modified unrelated files: succeeds, lands the worktree's commits, restores WIP to index, no work lost.
- Force a failure mid-dance (e.g., merge conflict); confirm `git reset --hard ORIG_HEAD` restores the pre-land state cleanly.

## What we're explicitly NOT doing yet

- Cross-branch overlap detection (file conflict warnings before merge).
- Auto-cleanup of abandoned worktrees.
- Recipe templates / pods / manager-evaluator pattern (h2pp absorption).
- Auto-cd or auto-spawn on task start.
- Replacing tmux.
- LLM-based topic-drift detection (cry-wolf risk; deterministic gates only).
